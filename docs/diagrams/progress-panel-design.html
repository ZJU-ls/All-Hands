设计要点

满足你的反馈
  • 不在顶部 · 在 InputBar 上方
  • plan 和 subagent **各自独立**可收起
  • 收起后 header 仍显示状态摘要
  • 动态 · 实时跟随状态变化(plan 步骤打勾、subagent 完成秒数累加)
  • 空状态隐身(简单聊天不打扰)
权衡
  • 跟 InputBar 抢一点垂直空间(plan + subagent 都展开 ~180px)
  • 但用户可两个都收起 · 占用降到 ~80px(两条 header)
  • 移动端 plan 步骤名长可能 truncate · 加 title attr

实现路径(确认后开工)

新组件

  • 新 web/components/chat/ProgressPanel.tsx ~80 LOC · 容器 · 决定整个 panel 是否显示
  • 新 web/components/chat/PlanProgressSection.tsx ~120 LOC · plan 折叠 section · 拉 plan 数据 · 渲染步骤
  • 新 web/components/chat/SubagentProgressSection.tsx ~100 LOC · 从 store 派生活跃 subagent · 渲染列表

数据源(都是动态/实时)

  • plan:GET /api/conversations/{id}/plans/latest(新轻量 endpoint)· 加上 SSE/store 触发的 invalidation —— 当 ToolMessageCommitted 来自 plan_create/update/complete 时 refetch · 几行代码
  • subagent:纯前端 · 从 useChatStore 派生 · 扫所有 message.tool_calls 找 dispatch_employee/spawn_subagent · running 的进 active list · done 的进历史(默认隐藏 · 展开看) · 实时跟着流式更新

展开状态持久化

  • 每个 section 的展开/收起状态存 localStorage · key=progress.plan.expanded / progress.subagent.expanded
  • 切换 conversation 不重置 · 用户偏好跨会话
  • 默认值:plan 展开 · subagent 展开

布局插入点

// web/app/chat/[conversationId]/page.tsx ~278

  ...

实时刷新机制

  • plan section:initial fetch + 每次 ToolMessageCommitted 含 plan_id 时增量 refetch (debounce 200ms)
  • subagent section:zero-cost · pure derive from store (zustand selector)· 流式自动刷
  • 计时器(running 时长)· setInterval 1s 刷一下 metadata
