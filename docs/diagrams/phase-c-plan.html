§ 2 三个特性 · 简化版

C1 · Plan Tool 补齐 基础已有 缺 executor

Plan 是 agent 的内部 memo · scope=WRITE / requires_confirmation=False(不动外部系统)· 用户在 chat 流里通过 PlanTimeline 看到进度。

需要做什么

  • 新文件 execution/tools/meta/plan_executors.py · 4 个 executor(create / update_step / complete_step / view)
  • plan_view 返回 render envelope · component=PlanTimeline · props 含 steps
  • AgentLoop ctor 加 plan_repo + conversation_id 参数 · _maybe_substitute_executor 加 4 分支
  • ChatService _build_runner_factory 注入 plan_repo + conversation_id

不做什么

  • ❌ 没有 plan mode · 没有 conversation.permission_mode 字段
  • ❌ 没有权限 gating · 任何时候 agent 都能调 plan tools
  • ❌ 没有 enter/exit 仪式 · 直接调用就行

C2 · Sub-Agent Stream + 详情 Drawer drawer 基础已有 缺流式 + 按钮

子 agent 实时 yield 事件到父流 · ToolCallCard 加按钮 · 点击通过 ?trace=<run_id> 弹现有右侧 drawer。

后端流式

  • Tool 模型加 flag streams_events: bool · dispatch / spawn 设为 True
  • InternalEvent 加 optional subagent_id field
  • AgentLoop body 检测 streams_events:executor 是 async generator · yield-through 子事件
  • 子 result 里塞 run_id(子 run 的 trace 锚)

前端按钮

  • ToolCallCard expand 区检测 result.run_id · 渲染"查看子代理细节 →"按钮
  • 点击 → router.push(?trace=<run_id>) · 现有 RunTraceDrawer 监听并打开
  • RunTracePanel 已能 fetch + 渲染 · 不动

C3 · Clarification (ask_user_question) 全新

唯一全新的能力 · 复用 ADR 0018 留好的 DeferredSignal 原语 · 是 ConfirmationDeferred 之后的第二个 impl。

新建

  • core UserInput + UserInputRepo + alembic migration
  • UserInputDeferred(DeferredSignal) · publish/wait 模式
  • tool ask_user_question · 新 flag requires_user_input=True
  • HTTP POST /api/user-input/{id}/answer
  • web UserInputDialog.tsx · 多问题表单

扩展

  • AgentLoop _permission_check · requires_user_input 走 Defer(UserInputDeferred)
  • tool_pipeline:outcome.kind == "answered" 时把 payload(answers dict)merge 进 block.input 再 invoke executor
  • ag_ui_translator 加 allhands.user_input_required custom event
