原理 · 一图看清

user
"帮我重构 X 模块"
agent
调 plan_create(title="重构 X", steps=["读代码", "拆模块", "测试"])

agent
调 plan_view() · 返回 {component: "PlanTimeline", props: {steps: [...]}}

user
看到时间线第一项打勾 · 第二项 running · 心里有底

🧪 怎么测

方式 A · API 直接验证 plan tool 持久化
# 假设已有一个 conversation_id (从 /api/conversations 拿)
CONV=$(curl -s -XPOST http://localhost:8000/api/conversations \
  -H "Content-Type: application/json" -d '{"employee_id":"emp-lead"}' | jq -r .id)

# 让 agent 触发 plan_create:发一条消息
curl -N http://localhost:8000/api/conversations/$CONV/messages \
  -H "Content-Type: application/json" \
  -d '{"content":"帮我做一个三步骤的重构计划,然后用 plan_view 给我看看","thinking":false}' \
  | head -100  # 看 SSE 流里有 PlanTimeline render envelope
方式 B · 浏览器交互
  1. 打开 chat 页 · 选一个 employee(Lead 或任何带 plan_* 工具的)
  2. 输入: "帮我列出三个步骤的代码重构计划,做完用 plan_view 展示给我"
  3. 观察: agent 调 plan_create · 然后 plan_view · chat 流里出现 PlanTimeline 卡片(三步 · 全 pending)
  4. 追问: "开始第一步,完成后再 plan_view 一次"
  5. 观察: agent 调 plan_update_step(0, "running") → plan_complete_step(0) → plan_view · 看到第一项打勾
