01

列表 / 网格视图切换

component · view-mode
Why
不同任务需要不同密度。「找一个旧产物快速点开」适合密集列表；「浏览审视团队最近的输出」适合带类型图标的画廊卡片。一个视图喂不饱两类心智模型。
参考
Notion · gallery view Figma Community · cards Anthropic Files GitHub Files
实现
  • 新组件 ArtifactGrid · 1/2/3 列响应式 · kind-tinted icon tile（markdown/code → primary, html → warning, image/csv/xlsx → success, pdf → danger）+ name + meta（version · size · 相对时间）+ 置顶角标
  • 页面 localStorage-backed view mode · 用户的密度偏好跨访问保留
  • 布局自适应：list → 侧栏 4/12 + 详情 8/12 · grid → 画廊 8/12 + 详情 4/12 · 没选中时画廊吃满 12 列

04

Hover 浮窗预览（Peek）

component · interaction
Why
扫一眼想知道详情，但不想为每张卡都点开右侧详情面板。Quick Look 心智 — hover 短停看一眼，确认要的再点。
参考
macOS Finder · Quick Look GitHub · linked-issue hover-card Notion · mention preview Vercel · deployments popover
实现
  • ArtifactPeek render-prop wrapper · 220ms intent delay（更短会闪、更长会滞）
  • 位置算一次（rect.right + 12px），右边不够时 flip 到左边 · 不会 mid-hover 抖动
  • 展示：kind icon + name + version + size + updated/created relative time + conversation 关联
  • 触屏（pointer: coarse）自动跳过 — 触屏没有真正的 hover 概念，强制弹会让用户摸不着头脑
  • 挂载在 ArtifactGrid（视觉浏览场景最受益），List 视图本身已有足够 meta

骨架屏 + 可操作的空态

perceived perf · empty states
Why
居中转圈让用户每次切筛选都要等到结果"砰"地出现 — 加上 shape-aware 骨架屏，chrome 不动，结果就位。空态分两种心智，CTA 也得分两种。
参考
Vercel · deployments skeleton GitHub · Files skeleton Linear · empty list CTA
实现
  • 新组件 SkeletonList · 跟随 viewMode：list 模式 6 行；grid 模式 6 卡片。形状跟最终内容一致，结果落位时 chrome 不抖
  • EmptyList 分 2 路：filtered → 0 显示 "Clear filters" CTA · workspace empty 显示静态指引文案
  • "Clear filters" 复用 chip-strip 的同一个 handler — 单一 source of truth
