主线索四 · 表单交互

05DesignForm TextField 加内联校验 feat

之前必填字段只有 label 红点 *,用户填错只能在 save 之后看通用 toast,体验不连贯。现在:

  • onBlur 标记 touched,空值显红框 + "必填字段不能为空"
  • 新 hint(灰)/ error(强)槽位让后续 form 复用同一模板
  • aria-invalid 同步,a11y 工具能感知

06Composer textarea autosize feat

之前 rows=2 写死,用户输 200 字段落时光标"掉出框外"。现在每次 value 变就重测 scrollHeight:贴合内容高度;到 ~8 行(≈ 188px)触顶后 overflow-y auto 接管。对照 ChatGPT / Claude.ai 体验。

Why: 这是聊天为先的产品,composer 是核心面;固定 2 行让长 prompt 写起来像在控制台编辑命令,而不是和 AI 对话。

主线索五 · 语义清晰 / 概念解耦(前传)

10"上下文窗口"字段消歧 + composer chip tooltip 说清概念 copy

用户 raise:gateway 配的 64k 是"最大输出 tokens"语义,而 chat composer 把它当输入预算分母用,概念错位。

  • Gateway "上下文窗口"字段加显式 hint:"模型一次能处理的 token 总额(prompt+completion),chat 预算条以此为分母,不是最大输出"
  • Composer chip tooltip 改成"已用输入估算 N tokens / 模型上下文窗口 M tokens"

11拆分 max_input_tokens / max_output_tokens 为独立字段 feat

把"上下文窗口"单字段不合理压成的三个概念拆成三个独立可选字段:

  • context_window — 总额(prompt + completion)
  • max_input_tokens — composer 预算条分母 · 真实 prompt 上限
  • max_output_tokens — 出站 chat 的 max_tokens · LangChain ctor 时烘到 payload(Anthropic 的 max_tokens 和 thinking 一样,bind 不传递)

落地:core/model.py + alembic 0028 + persistence + service + api/routers + meta tools schema + execution/llm_factory(ctor-time 注入) + chat_service(根据 LLMModel.max_output_tokens 注入 AgentLoop) + 前端 ModelDto + Gateway form(收进新建的 <Collapsible> 高级设置区,默认收起) + composer UsageChip(分母优先用 max_input_tokens)。

12注册模型表单 · 高级设置折叠 + 滚动 tip feat

注册时只必填 name + display_name。三个 token cap 收进折叠的"高级设置"区,默认收起;编辑时若已填过则自动展开。

顶部 <RotatingTip> 4.5s 轮播 4 条文案告诉用户:caps 可选 / Lead Agent 也能配 / ctx vs max input 的区别。两条思路并存:UI 自助 + 对话式配置。
