## 0 · TL;DR

- **触发器 = 把零散 Agent 员工拼成自运转工作流的关键机制**
- 两种触发形态:**Timer**(cron)· **Event**(pub/sub 订阅 artifact / run / confirmation / webhook 等事件)
- 4 种 action:`dispatch_employee` / `continue_conversation` / `invoke_tool` / `notify_user`
- **必须**的防爆规则:min_interval / 连续失败 auto-disable / 嵌套防环 / 全局限额 / pause-all
- 完整的 Meta Tool 族(创建 / 启停 / 手动 fire / 查看历史)· 驾驶舱有 UI · Lead Agent 也能对话操作

## 1 · 问题陈述

[agent-design.md](./2026-04-18-agent-design.md) 设计了员工本体 + dispatch,但**没人主动 dispatch 的时候,员工就是一堆静物**。要让系统真正成为"AI Team":

- **定时**:每天 8am 让 researcher 写日报
- **事件响应**:PR 合并(artifact.updated)→ 自动让 writer 写 release note
- **链式**:trigger A 产物触发 trigger B → 业务流自动串起来

触发器是这三种"自运转"的统一入口。缺了它,用户得手动去 chat 敲一次才跑一次,数字员工组织就变成数字问答机。

## 2 · 原则

### 2.1 统一模型 · 两种形态

一个 `Trigger` 实体,`kind: Literal["timer", "event"]` 分形态。数据模型共用,只是触发源不同。**禁止**两张表两套 service。

### 2.2 Action 是"做什么"的一等抽象

action 不是一行 hack。它是 `TriggerAction` 联合类型:

- `dispatch_employee`(新起会话跑员工)
- `continue_conversation`(追加消息到指定会话)
- `invoke_tool`(直接调单个 tool 做一次副作用)
- `notify_user`(只提醒,不干活)

这 4 种覆盖 v0 所有自运转场景。更多留给 v1。

### 2.3 防爆比功能重要

触发器是平台内**最容易失控**的组件(一个不好做法 = 无限循环 = 账单爆炸)。本 spec **必须**实装 5 条防爆规则(§ 7),缺一不可。

### 2.4 Tool First · UI 能做的 Lead 也能做

创建 / 启停 / 手动 fire / 看历史 / 删除 → 每一个 UI 按钮都有对应 Meta Tool。Lead Agent 在对话里说"把每日日报触发器停一下"能直接执行。

## 3 · 数据模型

### 3.1 `Trigger`(`core/trigger.py`)

```python
class TriggerKind(str, Enum):
    TIMER = "timer"
    EVENT = "event"

class TriggerAction(BaseModel):
    type: Literal[
        "dispatch_employee",
        "continue_conversation",
        "invoke_tool",
        "notify_user",
    ]
    # dispatch_employee
    employee_id: str | None = None
    task_template: str | None = None
    # continue_conversation
    conversation_id: str | None = None
    message_template: str | None = None
    # invoke_tool
    tool_id: str | None = None
    args_template: dict[str, Any] | None = None
    # notify_user
    channel: Literal["cockpit"] | None = None
    message: str | None = None

class TimerSpec(BaseModel):
    cron: str                        # 标准 5 段
    timezone: str = "UTC"            # IANA

class EventPattern(BaseModel):
    type: str                        # 见 § 4.2 枚举
    filter: dict[str, Any] = {}      # 简单 key-equality 或 glob

class Trigger(BaseModel):
    id: str                          # ulid
    name: str
    kind: TriggerKind
    enabled: bool = True

    timer: TimerSpec | None = None       # kind=timer 时必填
    event: EventPattern | None = None    # kind=event 时必填

    action: TriggerAction
    min_interval_seconds: int = 300      # 防爆:默认 5 min

    # 运行时统计
    fires_total: int = 0
    fires_failed_streak: int = 0
    last_fired_at: datetime | None = None
    auto_disabled_reason: str | None = None  # 被 auto-disable 时写明原因

    created_at: datetime
    created_by: str                      # "system" / "lead_agent" / user
```

### 3.2 `TriggerFire`(一次触发的记录 · 持久化)

```python
class TriggerFire(BaseModel):
    id: str
    trigger_id: str
    fired_at: datetime
    source: Literal["timer", "event", "manual"]
    event_payload: dict[str, Any] | None  # event 源才有
    action_snapshot: TriggerAction        # 触发时的 action 定义(之后改了不影响)
    rendered_task: str | None             # 模板渲染后实际下发的指令
    run_id: str | None                    # dispatch_employee / continue_conversation 产生的 run
    status: Literal["queued", "dispatched", "succeeded", "failed", "suppressed"]
    error_code: str | None
    error_detail: str | None
```

### 3.3 迁移 · `0005_add_triggers.py`

```sql
CREATE TABLE triggers (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    kind TEXT NOT NULL,                  -- timer/event
    enabled BOOLEAN NOT NULL DEFAULT TRUE,
    timer JSONB,                         -- {cron, timezone}
    event JSONB,                         -- {type, filter}
    action JSONB NOT NULL,
    min_interval_seconds INTEGER NOT NULL DEFAULT 300,
    fires_total INTEGER NOT NULL DEFAULT 0,
    fires_failed_streak INTEGER NOT NULL DEFAULT 0,
    last_fired_at TIMESTAMPTZ,
    auto_disabled_reason TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_by TEXT NOT NULL,
    workspace_id TEXT NOT NULL
);

CREATE TABLE trigger_fires (
    id TEXT PRIMARY KEY,
    trigger_id TEXT NOT NULL REFERENCES triggers(id) ON DELETE CASCADE,
    fired_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    source TEXT NOT NULL,
    event_payload JSONB,
    action_snapshot JSONB NOT NULL,
    rendered_task TEXT,
    run_id TEXT,
    status TEXT NOT NULL DEFAULT 'queued',
    error_code TEXT,
    error_detail TEXT
);
CREATE INDEX idx_trigger_fires_trigger ON trigger_fires(trigger_id, fired_at DESC);
```

## 4 · Event Bus · 事件总线

### 4.1 `events` 表(与 [cockpit.md § 7](./2026-04-18-cockpit.md#7) 共享)

**同一张表**,哪份 spec 先交付,migration 就放到哪份。两份 spec 共用 schema,见 cockpit § 7。events 表是触发器 + cockpit 活动流 + observatory 事件投影的公共底座。

### 4.2 可订阅的事件类型(pub/sub 枚举)

这是 **协议**,执行端必须照这份表实现发布点:

| event kind | 发布方 | payload |
|---|---|---|
| `artifact.created` | artifacts-skill | `{id, kind, name, run_id}` |
| `artifact.updated` | artifacts-skill | `{id, kind, name, version, diff?}` |
| `artifact.deleted` | artifacts-skill | `{id, kind, name}` |
| `run.started` | AgentRunner | `{run_id, employee_id, trigger_id?, parent_run_id?, depth}` |
| `run.completed` | AgentRunner | `{run_id, status, duration_s}` |
| `run.failed` | AgentRunner | `{run_id, error_code, detail}` |
| `confirmation.requested` | Gate | `{confirmation_id, tool_id, scope, conversation_id}` |
| `confirmation.resolved` | Gate | `{confirmation_id, decision: approved/denied, by}` |
| `confirmation.expired` | Gate | `{confirmation_id, tool_id}` |
| `mcp.unreachable` | MCPClient | `{server_id, retry_count}` |
| `employee.created` | EmployeeService | `{employee_id, name}` |
| `webhook.external` | `POST /api/webhooks/{webhook_id}` | `{body, headers}` |

每个发布点:
1. 写入 `events` 表(持久化,给 cockpit 活动流用)
2. 发布到进程内 pub/sub(给 trigger executor + SSE stream 用)

两步合成一个 `EventBus.publish(kind, payload)` 调用。实现方:`execution/events.py`。

### 4.3 EventPattern 匹配规则

`EventPattern.type` 严格 **==** event kind。`filter` 支持:

- 直接字段相等(`{"employee_id": "writer"}`)
- 字符串 glob(`name_pattern: "**/CHANGELOG*"` → fnmatch)
- 本 spec v0 **不支持** 表达式语言

## 5 · Trigger Executor

### 5.1 Timer 调度器

`execution/triggers/timer_scheduler.py`:
- 使用 **APScheduler**(Python 库,已被 FastAPI 生态广泛验证)启动一个 `AsyncIOScheduler`,挂在 FastAPI lifespan 里
- 启动时从 DB 加载所有 `kind=timer & enabled=true` 的 trigger,注册到 scheduler
- 创建 / 启停 / 删除 trigger 时同步更新 scheduler
- 触发时调用 `TriggerExecutor.fire(trigger, source="timer")`

### 5.2 Event 监听器

`execution/triggers/event_listener.py`:
- 订阅 `EventBus`,收到事件后扫描所有 `kind=event & enabled=true` trigger,匹配 `EventPattern`,命中则 `TriggerExecutor.fire(trigger, source="event", event_payload=payload)`
- 匹配逻辑 < 1ms(内存里扫一遍 trigger 列表,v0 规模 < 100 trigger)

### 5.3 `TriggerExecutor.fire()` 流程

```python
async def fire(trigger, source, event_payload=None) -> TriggerFire:
    # 1. 防爆 · § 7 五道关一个都不跳
    if not passes_rate_limit_per_trigger(trigger): return suppressed("rate_limit_per_trigger")
    if not passes_global_rate_limit(): return suppressed("global_rate_limit")
    if detects_cycle(trigger, event_payload): return suppressed("cycle")

    # 2. 渲染模板
    rendered = render_template(trigger.action, ctx={
        "trigger": trigger, "event": event_payload, "timer": {...},
        "@yesterday": ..., "@today": ..., "@last_run": ...,
    })

    # 3. 落一条 TriggerFire(status=queued)
    fire = await persist_fire(...)

    # 4. 分派到 action handler
    try:
        run_id = await ACTION_HANDLERS[trigger.action.type](rendered, trigger_id=trigger.id)
        fire.status = "dispatched"; fire.run_id = run_id
    except Exception as e:
        fire.status = "failed"; fire.error_code = code(e); fire.error_detail = str(e)
        trigger.fires_failed_streak += 1
        if trigger.fires_failed_streak >= 3:
            trigger.enabled = False
            trigger.auto_disabled_reason = f"3 consecutive failures: {fire.error_code}"
            await emit_alert(trigger)
    else:
        trigger.fires_failed_streak = 0
    finally:
        trigger.fires_total += 1
        trigger.last_fired_at = fire.fired_at
        await persist_trigger(trigger)
        await persist_fire(fire)
    return fire
```

### 5.4 Action handlers

写在 `execution/triggers/actions/*.py`:

- `dispatch_employee`:调 `run_service.start_run(employee_id=..., task=rendered.task, trigger_id=trigger.id)`(是 agent-design.md § 6 的 dispatch 入口 · 但 trigger 路径下 `parent_run_id=None`,`trigger_id` 作为新字段标识触发来源)
- `continue_conversation`:调 `conversation_service.append_message(conversation_id, rendered.message)`,然后 `run_service.continue_conversation(conversation_id)`
- `invoke_tool`:直接调 `tool_registry.get(tool_id).execute(args)`;**如果 scope >= WRITE 要 requires_confirmation**,则 trigger 层拒绝:"触发器不能直接触发写操作 tool,用 dispatch_employee 让 agent 带上下文去调"
- `notify_user`:写一条 `ActivityEvent(kind="trigger.notify", severity=...)` 到 events 表,cockpit 展示

## 7 · 防爆规则(必须全部实装,漏一个 = 本 spec 不通过)

### 7.1 每触发器限速 · `min_interval_seconds`

- 默认 300s(5 min)· 用户可改,**最小允许 60s**
- 检查方式:`if now() - last_fired_at < min_interval` → `suppressed("rate_limit_per_trigger")`
- 不是"队列等到时间到",是**直接丢弃**。触发风暴期要快速返回

### 7.2 连续失败 auto-disable

- `fires_failed_streak >= 3` → `enabled = False` + `auto_disabled_reason = "..."` + 发 `activity: trigger.auto_disabled` 事件
- 用户 / Lead 手动 enable 回来时 streak 清零

### 7.3 嵌套防环

`TriggerFire` 产生的 run 如果又产生事件,该事件不得**直接**触发同一 trigger:

- `run.started` 事件带 `trigger_id`(§ 4.2 schema 已含)
- event listener 匹配时:如果 `event_payload.trigger_id == trigger.id` → `suppressed("cycle")`
- 并且:follow-up 事件(run 产的 artifact)也通过 `run_id` → `trigger_id` 反查出"祖先 trigger 链",出现自己在链上 → suppressed

### 7.4 全局限额 · `MAX_TRIGGER_FIRES_PER_MINUTE`

- env 默认 60。滑动窗口计数
- 超过 → 所有触发暂停,emit `trigger.global_rate_limit` alert。1 分钟后自动恢复。期间 Lead 可 `cockpit.pause_all_runs()` 彻底停

### 7.5 全局开关 · `triggers_paused`

- workspace-level 开关(env + DB · 都支持)
- `cockpit.pause_all_runs` 会把它切到 `true`(同时 cancel 现有 run)
- 所有 trigger executor 起手先查:paused → suppressed

## 12 · 开放问题 · Decision defaults

1. **Q**: webhook 要不要签名验证?
   **Default**: v0 每个 webhook-trigger 生成一个随机 token,URL 路径内(`/api/webhooks/{id}?token=...`)。v1 上 HMAC header。

2. **Q**: timer 能不能设"一次性"(不 cron,只在某个时刻)?
   **Default**: v0 不做,用 cron 表达式 + 手动删除。v1 加 `one_shot: bool`。

3. **Q**: min_interval 最小值 60s 够不够?
   **Default**: 够。更小的频率(秒级)说明用户在做实时事件响应 → 应该走 event 而不是 timer。

4. **Q**: 外部 webhook payload 大小限制?
   **Default**: 256 KB,超过拒绝。v0 不做大 payload 场景。

5. **Q**: 事件订阅同一个 event 命中多个 trigger,执行顺序?
   **Default**: 无保证 · 并发启动。用户在 UI / spec 里不得依赖顺序。
