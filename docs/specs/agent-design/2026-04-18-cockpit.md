## 1 · 问题陈述

今天的 `/` 是"打开就掉进 Lead 聊天"。但 allhands 的定位是 **"一个人驾驶一支 AI Team"**。用户进门应该站在**驾驶位**,一眼看到:

- 谁在干活(哪些 run 正在跑)
- 出了什么事(活动流)
- 系统稳不稳(gateway / MCP / langfuse / db / triggers)
- 今天花了多少钱(token / cost 预算感)
- 下一步能立刻做什么(新对话 / 新员工 / 新触发器 / 急停)

聊 Lead 变成 **快速操作** 之一,而不是默认落脚点。用户把焦点从"和一个 agent 说话"换成"看一整支团队的实时状态"。

## 2 · 原则

### 2.1 全知

Cockpit 一屏要覆盖平台内所有"动着的东西"的当下快照:employees / runs / conversations / triggers / artifacts(本周增量)/ tokens / 健康。**单屏** + **可下钻**:任何卡片都能点进对应详情页。

### 2.2 全控

关键干预必须在首页可触达:
- 新对话 / 新员工 / 新触发器(创建入口)
- kill 单个 run(进 run 卡片)
- 全局暂停(顶层 pause-all)
- 确认等待队列(confirmation 堆积时首页高亮)

### 2.3 实时

所有活动数据由一条 SSE 流推送。不 poll。SSE 断 → 自动重连 + 顶部 banner 告警。

### 2.4 Tool First · 驾驶舱能做的,Lead Agent 也能做

符合 L01 Tool First 扩展版:
- UI 能看到的快照 → `cockpit.get_workspace_summary()`(READ Meta Tool)
- UI 能按的"pause all" → `cockpit.pause_all_runs()`(IRREVERSIBLE Meta Tool · 要 confirmation)

Lead 在对话里可以回答:"现在系统里在跑什么?""先把所有 run 暂停一下"。

## 3 · 数据模型(新增最小化)

驾驶舱不引入新的持久化表。它是一个**聚合视图**,数据全部来自已有表 + 事件总线。

### 3.1 `WorkspaceSummary` DTO(`core/cockpit.py`)

```python
class WorkspaceSummary(BaseModel):
    # 顶部 KPI
    employees_total: int
    runs_active: int
    conversations_today: int
    artifacts_total: int
    artifacts_this_week_delta: int
    triggers_active: int
    tokens_today_total: int          # prompt + completion
    tokens_today_prompt: int
    tokens_today_completion: int
    estimated_cost_today_usd: float  # 粗算,按 model 单价表

    # 健康
    health: HealthSnapshot           # 见 § 3.2

    # 队列
    confirmations_pending: int       # 等待用户确认的 tool call 数
    runs_failing_recently: int       # 近 1h failed

    # 最近 N 条活动
    recent_events: list[ActivityEvent]   # § 3.3,取最近 20 条
    active_runs: list[ActiveRunCard]     # § 3.4,取全部(一般 ≤ 10)
    recent_conversations: list[ConvCard] # 取最近 5
```

### 3.2 `HealthSnapshot`

```python
class HealthSnapshot(BaseModel):
    gateway: ComponentStatus         # ok/degraded/down + 4/4
    mcp_servers: ComponentStatus     # ok + 3/3
    langfuse: ComponentStatus        # ok/paused
    db: ComponentStatus              # ok
    triggers: ComponentStatus        # ok · 或 "1 paused"
```

每一项都是 `{ name, status: "ok"|"degraded"|"down", detail: str }`。

### 3.3 `ActivityEvent`(活动流的一条)

```python
class ActivityEvent(BaseModel):
    id: str                          # ulid
    ts: datetime
    kind: Literal[
        "run.started", "run.completed", "run.failed",
        "artifact.created", "artifact.updated",
        "trigger.fired", "trigger.bound",
        "confirmation.pending", "confirmation.resolved",
        "mcp.unreachable",
    ]
    actor: str | None                # employee_id / trigger_id / system
    subject: str | None              # run_id / artifact_id / ...
    summary: str                     # "coder 在写 /web/app/..."
    severity: Literal["info", "warn", "error"]
    link: str | None                 # 点一下跳去哪(/chat/{id} / /artifacts/{id} / ...)
```

活动流是 **events 表的投影**(事件总线统一写入一张 `events` 表,cockpit 读最近 N 条)。events 表由 triggers spec § 4.1 定义,两份 spec 共用。

### 3.4 `ActiveRunCard`

```python
class ActiveRunCard(BaseModel):
    run_id: str
    employee_id: str
    employee_name: str
    status: Literal["thinking", "calling_tool", "waiting_confirmation", "writing"]
    current_action_summary: str      # "调用 write_file(/web/app/...)"
    iteration: int
    max_iterations: int
    parent_run_id: str | None        # 嵌套 run 时,父
    depth: int                       # 0 = 根
    started_at: datetime
    trigger_id: str | None           # 由 trigger 触发则填
```

## 4 · 后端 API

### 4.1 `GET /api/cockpit/summary` · 快照(非 stream)

返回 `WorkspaceSummary`。给 **初次进页 + 刷新页面** 用。2026-04-18 实现就够精度:

- employees_total / runs_active / triggers_active:直接 count
- conversations_today / tokens_today_*:过去 24h
- artifacts_this_week_delta:按本周起算
- estimated_cost_today_usd:每次 run 完成时按 model 单价算增量,存 `runs.estimated_cost`,汇总

### 4.2 `GET /api/cockpit/stream` · Workspace-level SSE

**这是本 spec 引入的最关键 endpoint**(符合 [architecture.md § L8](../../../product/04-architecture.md) 的 SSE 扩展)。一条流订阅 workspace 内所有事件。

**事件类型**(与 events 表一致):

| SSE `event:` | payload | 前端做什么 |
|---|---|---|
| `snapshot` | 完整 `WorkspaceSummary` | 首次连接 / 重连后整体刷新 |
| `activity` | 单条 `ActivityEvent` | 插到活动流顶部 |
| `run_update` | 单个 `ActiveRunCard` | 中列 run 卡片原地更新 |
| `run_done` | `{run_id}` | 从"正在执行"移除 |
| `health` | `HealthSnapshot`(整体) | 右列替换 |
| `kpi` | `{key, value}` 单个 KPI 变 | 顶部 counter 动一下 |
| `heartbeat` | `{ts}` | 每 15s 一次,心跳 |

**连接开始时服务端立刻 push 一条 `snapshot`**,然后增量。

### 4.3 `POST /api/cockpit/pause-all`

- body: `{reason?: str}`
- 行为:workspace 内所有 `runs_active` 发 cancel signal,同时暂停 trigger executor
- 要求 `X-Confirmation-Token`(走 ConfirmationGate 的标准流程)
- 幂等:已暂停状态下再调 → 200 + `already_paused=true`

### 4.4 `POST /api/cockpit/resume-all`

对称。无需 confirmation(只是让系统恢复"能干活"的状态)。

## 5 · Meta Tools(给 Lead Agent 用)

写在 `execution/tools/meta/cockpit.py`,注册进 `ToolRegistry`。

### 5.1 `cockpit.get_workspace_summary`(READ)

```python
Tool(
    id="allhands.meta.cockpit.get_workspace_summary",
    name="cockpit.get_workspace_summary",
    scope=ToolScope.READ,
    requires_confirmation=False,
    description=(
        "Get a single-snapshot summary of the workspace: how many employees, "
        "how many runs currently active, today's token usage, system health "
        "(gateway/MCP/langfuse/db/triggers), recent activity (last 20 events), "
        "and any runs currently waiting for user confirmation. "
        "Use this when the user asks questions like 'what's running right now' "
        "or 'is everything healthy' before answering."
    ),
    input_schema={},  # 无参
    output_schema=<WorkspaceSummary json-schema>,
)
```

### 5.2 `cockpit.pause_all_runs`(IRREVERSIBLE)

```python
Tool(
    id="allhands.meta.cockpit.pause_all_runs",
    name="cockpit.pause_all_runs",
    scope=ToolScope.IRREVERSIBLE,
    requires_confirmation=True,       # 走 Gate
    description=(
        "Emergency brake: cancel every currently-active run and pause the "
        "trigger executor. Active runs get a cancel signal; in-flight tool "
        "calls may not roll back. Call ONLY when the user explicitly asks "
        "to stop everything, or when you detect a runaway loop. This is "
        "IRREVERSIBLE as far as in-flight side effects are concerned."
    ),
    input_schema={
        "reason": {"type": "string", "description": "Why pausing; shown on cockpit."}
    },
)
```

**不**提供 Meta Tool `kill_run(run_id)` —— 单个 kill 属于 run 生命周期管理,留给 run API / 本 spec 后续迭代。v0 只给"雷管"。

### 5.3 Lead Agent system prompt 补丁

`backend/src/allhands/execution/prompts/lead_agent.md` 增加一段:

> **When the user asks about workspace state** — number of runs, what's active, whether something is broken, today's cost — **call `cockpit.get_workspace_summary` first**, then summarize. Don't try to piece it together from list_employees / list_triggers separately.

### 6.3 SSE 客户端

`web/lib/cockpit-stream.ts` 统一一个 EventSource(不是多个):

- 断线重连(指数退避,封顶 30s)
- 收到 `snapshot` → 全量替换 store
- 收到增量 → 合并到 store(zustand 或 redux 任选,本 spec 不强制)
- `heartbeat` 超时(> 45s 没收到)→ 主动重连

## 7 · 事件投影:events 表

> **本节与 [triggers.md § 4.1](./2026-04-18-triggers.md#41) 共享。**两份 spec **合并**成同一张 `events` 表(migration 放到先落地的那份)。

```sql
CREATE TABLE events (
    id TEXT PRIMARY KEY,             -- ulid
    ts TIMESTAMPTZ NOT NULL DEFAULT now(),
    kind TEXT NOT NULL,              -- "run.started" / "artifact.updated" / ...
    actor TEXT,
    subject TEXT,
    payload JSONB NOT NULL,          -- 完整事件体
    severity TEXT NOT NULL DEFAULT 'info',
    link TEXT,                       -- UI 下钻路径
    workspace_id TEXT NOT NULL
);
CREATE INDEX idx_events_ts ON events(workspace_id, ts DESC);
CREATE INDEX idx_events_kind ON events(workspace_id, kind, ts DESC);
```

- 每条事件 **同时** 写 events 表 + 发给内存中的 pub/sub(给 SSE 和 trigger executor 用)
- 保留策略:90 天后归档 / 删除(v1);v0 不清理
- Cockpit `/api/cockpit/summary` 的 `recent_events` 从这张表查最近 20 条

## 8 · Confirmation 堆积的特殊处理

当 `confirmations_pending > 0`,驾驶舱需要 **醒目** 展示:

- KPI bar 顶上增加 `⚠ {N} 待确认` 红字链接
- 点击 → `/confirmations`(一个简单列表;已有还是新建由 executor 看)
- 活动流中 `confirmation.pending` 事件用 `#d97706` 左条

本 spec **不自己**做 confirmations 页面细节。若现状没有 `/confirmations`,autopilot 可以临时把它降级成 Lead 会话里的 confirmation banner 聚合。

## 12 · 开放问题 · Decision defaults

1. **Q**: 活动流里"同员工连续 3 条"要不要折叠?
   **Default**: v0 不折叠,按时间完整列出;v1 按需。

2. **Q**: KPI bar 的 `estimated_cost` 哪里算?
   **Default**: 在 `execution/runner.py` 里 run 完成时按 `model.pricing`(env `MODEL_PRICING_JSON` 覆盖)算一次,写 `runs.estimated_cost`。cockpit 只 SUM。

3. **Q**: Lead Agent 调 `cockpit.pause_all_runs` 要走 confirmation 吗?对,但 confirmation 需要**人类**按。Lead 自己不能跳过。
   **Default**: Gate 默认策略已经是"IRREVERSIBLE → 要 human token"。本 spec 不改策略。

4. **Q**: workspace_id 现在是不是还是单租户 hardcoded?
   **Default**: 是 `default`(见现状)。本 spec 用同一个,不引入多 workspace。v1 再做。
