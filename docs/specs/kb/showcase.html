能做什么

把零散的资料聚成可检索的库;让任何员工通过 kb_search 主动查证;给授权员工开 WRITE,把对话沉淀回 KB。

多格式摄取

md / txt / pdf(含扫描件 OCR)/ docx / pptx / xlsx / csv / html / epub / 图片 / 音频(Whisper)/ 视频 / 代码 / URL 递归。每种格式 = 注册一个 Parser。

Hybrid 检索

SQLite FTS5(BM25)+ sqlite-vec(向量)+ RRF 融合 + 可选 reranker。每个 KB 可调权重 / top_k / reranker 开关。

Agentic 多轮

不内置"检索引擎",把检索拆成 Meta Tool。Agent 在 query 主循环里自己决定要 search 几次、要不要拉原文 — 完全对齐 P3 Pure-Function Loop。

权限闭环

写操作走 Confirmation Gate;无 grant 自动 Defer 到用户审批。Grant 可挂在 employee 也可挂在 skill,带 expiry。

引用回链

每个 chunk 保留 section_path / page / span。Agent 回答带 doc#2.3 渲染卡,点开看原文位置。

可观测

摄取状态机 → SSE 实时;每次 search 上 LangFuse trace,看 latency / hit-rate / 命中分布。检索回归套件挂 P@5 / MRR。

Meta Tool 矩阵

Tool Scope Confirm 说明
kb_list READ — 列 workspace 下所有 KB
kb_browse_collection READ — 按路径列子 collection + 文档

kb_read_document READ — 拉文档全文或指定 section / span
kb_render_citation READ — 在聊天里发 KB.Citation render envelope
kb_create_document WRITE 需要 新建文档 · 无 grant 走 Defer 申请
kb_update_document WRITE 需要 · 显示 diff overwrite / patch · 升 version
kb_move_document WRITE 需要 改 collection_id
kb_tag_document WRITE 需要 加/删 tags
kb_ingest_url WRITE 需要 抓 URL → parse → ingest · 支持递归
kb_delete_document IRREVERSIBLE 需要 软删 30 天
kb_grant_permission WRITE 需要 颁发/调整 grant
kb_reindex_document WRITE — 重跑 chunk + embed
kb_set_retrieval_config WRITE — 调权重 / top_k / reranker
kb_create WRITE 需要 新建 KB(workspace 维度)

Agentic 检索 · 真实对话流

不内置"检索引擎",检索是 stateless tool · agent 在主循环里自己驱动。

CASE A · 研究员模式 · kb-researcher skill
我去年做的那个 hybrid retrieval 调研笔记里说 reranker 收益是多少?
→ kb_search(kb="brain", q="hybrid retrieval reranker 收益", k=8)
← 5 候选 · 最高 0.84 (notes/reranker-benchmark.md)
→ kb_read_document(id="doc_abc", sections=["§2 results"])
根据笔记 notes#§2 · bge-reranker-base 把 P@5 从 0.61 提到 0.74(+13pt),代价是单次延迟 +120ms。如果开 bge-large 收益再 +3pt 但延迟翻倍。
CASE B · 沉淀模式 · kb-curator skill (有 grant)
把刚才整理的 KB 设计要点存进 Engineering Specs collection
→ kb_create_document(kb="brain", collection="engineering-specs", title="KB v0 设计要点", content=…, kind=md)
⏸ Confirmation Gate · "新建 1 个文档到 Engineering Specs · 24KB" · [Allow] [Deny]
← 已创建 doc_xyz · v1
→ kb_render_citation([chunk_xyz_1])
已沉淀到 engineering-specs/KB v0 设计要点 · 38 chunks 已索引,接下来 @researcher 就能搜到了。

交付路线 · 6 周

每个阶段都能落地用 — 不堆功能,先有 BM25 → 再加向量 → 再加 agentic。

M1 2 周 · 最小可用

把骨架立起来

  • ✦ core models + alembic
  • ✦ parsers: md / txt / pdf
  • ✦ chunker(heading + recursive)
  • ✦ FTS5 BM25 检索
  • ✦ Meta Tools: list / search / read_document / create_document
  • ✦ UI: 三栏 + 上传 + 简易 search
  • ✦ kb-researcher skill 上架
验收:能上传 100 篇 PDF · agent 能引用原文回答
M2 2 周 · 向量 + 多格式

让检索"懂语义"

  • ✦ sqlite-vec + ModelGateway.embed
  • ✦ docx / html / pptx / xlsx / csv / epub parser
  • ✦ Hybrid + RRF
  • ✦ Grant 模型 + 写 tools 全套
  • ✦ UI: GrantPanel / RetrievalPlayground / 摄取进度时间线
  • ✦ kb-curator skill 上架
验收:多格式覆盖 · grant 闭环 · 检索质量肉眼可见提升
M3 2 周 · Agentic + 高级

把质量做硬

  • ✦ bge reranker(本地 ONNX)
  • ✦ audio/video transcription · faster-whisper
  • ✦ image OCR
  • ✦ URL 摄取 + sha256 增量
  • ✦ 检索回归:固定 query · P@5 / MRR 不退化
  • ✦ LangFuse trace + ADR 收尾
验收:全格式 · 回归套件 · ADR 立项
