1 · 现状审计 · 单页堆了多少东西

在一个 viewport 里, 极端情况下能同时存在这些 UI 表面:

7 个真实痛点
  • 📊 看不清自己 KB 里有啥 · 大局信息被挤进 280px 窄卡
  • 💬 Ask 体验受限 · 满屏一半 · 多轮 / sources / composer 挤一起
  • ⚙ 设置是 modal · 切模型 / 跑 diagnose 都该满屏认真做
  • 📄 doc detail 是 drawer · 想"原文 + 分片并排"做不到
  • 🔗 没有可分享 URL · 搜了问了, URL 不变, 转给同事就丢
  • ↺ 切 KB 丢上下文 · Ask 对话清空但 URL 没变
  • 📱 窄屏崩溃 · toolbar wrap · sidebar 占太多
一句话诊断

把"KB 总览 / 文档管理 / 搜 / 问 / 设置 / 单文档详情" 6 个不同心智活动塞在一个 URL 里, 用 mode 字段在 React 里来回切, 就是密度过高的根源。

每个活动的"工具" / "上下文" / "正确视宽" 都不一样, 凑在一起就是相互妥协 · 谁都没拿到该有的舞台。

3 · 提案 · 三级 IA + 路由树

3 个层级 · 每级只回答一个问题 · URL 决定一切。

/knowledge                                  L1 · "我有几个 KB · 哪个值得进?"
├── /knowledge/new                          (modal · 不是 route)
└── /knowledge/[kbId]                       L2 · "这个 KB 我要干啥?" (默认 Overview)
    ├── /knowledge/[kbId]/docs              L2 tab · 文档管理
    ├── /knowledge/[kbId]/ask               L2 tab · 全屏对话
    ├── /knowledge/[kbId]/search            L2 tab · 全屏搜索
    ├── /knowledge/[kbId]/settings          L2 tab · 设置(满屏 · 4 sub)
    └── /knowledge/[kbId]/docs/[docId]      L3 · "这一份文档里有啥?"
L1

KB hub

卡片网格 · 每张卡是一个 KB · 名 / doc count / sparkline / 上次活跃 / 是否需要修复 一眼读完 · 点击进 L2。

L2

单 KB 工作区 · 5 tab

Overview(默认) / Documents / Ask / Search / Settings · 每个 tab 自己 URL · 全屏布局 · 不再 mode-multiplex。

L3

单文档页

替代当前 drawer · 满屏画布 + 320px side rail · Original ↔ Chunks 平级 tab · 支持 split view。

7 · 等你拍板的几个决策点

下面 5 条我各有倾向 · 但都可改 · 你定。

  1. 1
    Overview tab vs 直接进 Docs

    我推荐 Overview 默认 · 给用户摸底机会。也可以 smart default:第一次给 Overview · 之后记住上次的 tab。

  2. 2
    L2 chrome 是 tab bar 还是 sidebar nav

    Tab bar 更轻; sidebar nav 视觉更"工作区"。我倾向 tab bar · 5 个 tab 不算多。

  3. 3
    Ask thread 持久化 是否进本次

    我推荐 不进 · 它是单独 feature(需要 conversation 表)· 一并做范围太大。

  4. 4
    Settings 满屏页 vs 大 sheet drawer

    我倾向满屏页 · diagnose 子 tab 的三栏对比真的需要 width。

  5. 5
    Next.js parallel routes / intercepting routes

    把 doc 页同时支持"满屏"和"从 docs tab 浮起的 modal"。技术性优化 · 不是 v0 必需 · 可放 P4 之后单独评估。

8 · 落实标准 · 完成后用这 6 条对照

凡有一条没满足 · 都要继续修。

  • L1 任何 KB card 一眼能看到 · 名 + doc count + sparkline + 上次活跃 + 是否需要修复(stale embedding)
  • L2 任何 tab 内不再出现"4 张 sidebar 卡 + 主区 + drawer + modal"同框
  • L3 doc 页支持原文 + chunks 并排 · 点 chunk 直接在原文里高亮
  • 所有路由可分享 · URL 编码 active KB / docId / search query / ask thread
  • 浏览器后退按钮在每一级都符合直觉
  • 移动端 / 窄屏不再 wrap 出乱七八糟的工具栏
