1 · 11 轮迭代细节

每轮一卡 · 参考的产品 + 具体改动 + 你能在哪儿看到。

R1

Ask · 停止生成 + 重新生成

ChatGPT/Perplexity 风

流式中:Send 按钮变 停止(danger 色) · 点击 abort 当前流并保留已收到的部分。

完成后:答案旁出 重新生成(refresh icon) · 重发同一问题但新一次模型采样。

"新对话"按钮换 X icon · 三个动作视觉区分清楚。停止时 activity 中 running 项立即标 done · 不留 dangling 计时器。

R2

Ask · 智能自动滚动

ChatGPT 风

监听 scroll · 距底 >80px 视为用户滚开 · pinned=false。

流式 delta 只在 pinned=true 时滚 · 不打扰阅读上文。

非 pinned 时浮一颗按钮:↓ 新内容生成中(streaming)/ 滚到最新(idle)· 点击 smooth scrollTo 回 pinned。

R3

Ask · 引用 hover preview + 代码块复制

NotebookLM 风

CiteChip 组件 hover/focus 出 tooltip 卡 · 显原文 + score + section_path · 不点也能确认引用对不对。

CodeBlock 组件:```...``` 围起的 fenced 段渲染 pre + 头部带语言 + 复制按钮(NotebookLM/ChatGPT 风)。

R4

Ask · 相关追问推荐

Perplexity 风

后端新 POST /api/kb/{id}/follow-ups · 接收 question + answer · 返回 3 条 LLM 生成的下一个问题。

前端 done 帧后 fan-out 拉 · 仅最后一轮渲染 chip 行(过去轮渲染会乱)· 点击 = 直接发 follow-up · 走多轮 history。失败优雅返 [] · UI 不显示空行。

R6

Documents · 排序 + 网格/列表 双视图

Linear/Notion 风

URL 编码 ?sort=recent|oldest|name|size|chunks(默认最近更新)· ?view=grid|list 两态。

List 模式渲染表格:checkbox · 标题 · 状态 · 片段 · 大小 · 更新 · 标签 6 列。体积/片段排序方便定位"哪些 doc 最厚"。

R7

Doc 页 · TOC 大纲侧栏

Notion/GitBook 风

Original tab 改 grid: 200px 大纲 + 主区。buildOutline 从 chunks.section_path 抽 unique 节点 · 缩进按 ' > ' 层级。

点大纲项 = 自动滚原文到位 + 1.8s 高亮(复用 Chunks split-view 的 highlight 路径)· < lg 隐藏(移动端不挤画布)。

全局 / 聚焦 composer / search input · skips 当用户已在输入框(避免抢键)。

R11

答案区 · 轻量 markdown 渲染

流式安全

三件套覆盖了真实 LLM 答案 90% 的格式需求:

  • InlineMd · `code` / **bold** / *italic* 三种内联
  • CodeBlock · fenced 段独立渲染 + 复制
  • CiteChip · [N] 引用 hover preview

没接 marked 是有意 · 流式 token 不闭合的 ** 不会变成乱码 · 半行码块在收齐前显示成原文。
