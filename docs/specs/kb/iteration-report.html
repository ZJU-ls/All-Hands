  • • KnowledgeService.ask() · 检索 → 拼上下文 → LLM 调用
  • • 严格指令 prompt:每个事实必须 [N] 标注 · 不知道就说不知道

拖拽 + 批量上传

摩擦力 -80%

不用点按钮 · 拖一堆文件到页面 · 全屏出 "松手即添加到 X" · 实时进度条显示 N/M + 失败计数。SQLite 写锁约束下做了顺序入库,不并发,但每条独立 try/catch。

file input multiple · onDragOver/onDrop · UploadProgressStrip 在工具栏下面停留 4s 自清。
4

URL 抓取入库

"web clipper"

工具栏新加 🔗 抓 URL 按钮 · 弹模态 · 输 URL/标题/标签 · httpx fetch · 复用 HTML parser · 当前文档列表里就出现一条新文档。

  • • _derive_title_from_url URL 末段去 hyphen 当 fallback 标题

早上验收 · 5 分钟一项一项点

前后端服务都还在跑(localhost:3000/knowledge)· 按下表点一遍即可。

  1. a) 拖拽多个文件到页面
    期望:全屏蓝色虚线 outline + 中央"松手即添加到 X" 提示;松手后顶部出 N/M 进度条,完成后 4s 自清。
  2. b) 工具栏「🔗 抓 URL」
    输一个静态 HTML 页面 URL(blog post / 文档页 · 不要 SPA),抓完文档列表多一条 source_type=url 的卡。
  3. c) 工具栏「搜 / 问」分段切到「问」
    用知识库里有的内容提问 · 回答区出 LLM 文本 + 内联 [1] chip · 点 chip 滚到 sources 卡;没配 LLM provider 会报 422 + 友好提示。
  4. d) 点开任一文档 → 三 tab
    概览 / 原文(markdown 渲染) / 分片(N) · 分片卡有「跳原文」按钮 · 点击切到原文 tab + 该段高亮 1.8s。
  5. e) sidebar Tags 点一个标签
    文档列表只剩带该标签的项 · 列表头部出 #tag ✕ chip · 点 ✕ 清。
  6. f) hover 文档卡 → 左上 checkbox · 多选 → 批量删
    选 2-3 个 · 顶部出"选中 N · 取消 / 批量删除" · 删除后 confirm 一次再实删。
  7. g) 故意上传一个失败的(改后缀名 .pdf 但内容是 binary)
    卡变红 + state_error 文字 + 黄色「重试入库」chip · 点击触发 reindex(还会失败 — 但闭环跑通了)。
  8. h) 删掉所有 KB 看 onboarding
    主区出 4 步引导 · 第 1 步自动检测 provider · 第 4 步是 /employees 跳转。
Round 2 验收(8-12)
  1. i) Ask 流式 + 多轮(迭代 8)
    切到「问」· 输个问题 · 答案应该 逐字打出来 而不是空白等几秒;答完底部出"继续追问 …" composer · 输"那它和 X 比呢"应该沿用上一轮上下文回答。

    切到「问」模式 · 主区顶部应该出 4 个起手 chip(基于 KB 里 doc 标题/章节自动生成) · 点一条直接发问。
  3. k) "为什么排在这"(迭代 10)
    切到「搜」· 搜个有结果的词 · 每条 result 下面有 ▾ 为什么排在这 · 展开见 BM25/向量贡献条 + 查询词 chip(命中绿/未命中灰) · chunk 文本里命中词应该 黄底高亮。
  4. l) 批量打标签(迭代 11)
    hover 出 checkbox · 多选 2-3 个 doc · 顶部出 "标签…" 按钮 · 点开模态 · 点已有 tag 应该循环 + → − → · 三态;输入框写 "blog, 2026q2" → Apply · 文档卡 tag 应该都更新。
  5. m) toolbar 瘦身(迭代 12)
    顶 toolbar 应该只有 5 个控件:KB select / + / 搜问 toggle + 输入 / Upload split。"抓 URL" 在 Upload 右侧 chevron 下拉里;状态筛选已经搬到 Documents 视图 header 里。
