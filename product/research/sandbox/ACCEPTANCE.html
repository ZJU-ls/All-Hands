S1

整理桌面 / 目录归类

file-organize skill · Tier 0 · 验证 8 个 local.* 中的 5 个
核心 Phase 1+2 5 分钟

1️⃣ 用户对话

把 ~/allhands-test/desktop-demo 里的文件按文件类型整理一下,创建子文件夹放好。

2️⃣ Agent 应该这么做

  1. 激活 file-organize skill(看 system prompt 注入 ~5KB body)
  2. 调 local.list_dir 看目录里有啥
  3. 调 local.glob 按扩展名分组(**/*.{pdf,xlsx,jpg,...})
  4. 列出归档计划告诉用户:"要把 12 个 .pdf 移到 PDFs/,5 个 .xlsx 移到 Spreadsheets/..."
  5. 等用户 OK
  6. 调 local.bash mkdir -p 建子目录
  7. 逐文件 local.move
  8. summary:移了 N 个文件到 K 个目录

3️⃣ Confirmation 弹窗时机

  1. list_dir / glob:✅ allow(默认) · 不弹
  2. bash mkdir:⚠️ ASK 弹窗 · 三按钮:[允许这次] [总是允许 mkdir:*] [总是允许 local.bash]
  3. local.move:⚠️ ASK 弹一次后剩下连续都过(50 calls 内)

4️⃣ 前置造数据

cd ~/allhands-test/desktop-demo && touch a.pdf b.pdf report.xlsx data.csv pic1.jpg note.md

S2

用自然语言找文件

local.find_natural · 包装 macOS mdfind · 验证 Spotlight 集成
Phase 6 2 分钟 macOS 限定

1️⃣ 用户对话

我上周改过的所有 markdown 文件 · 列给我看看

2️⃣ Agent 应该这么做

  1. 调 local.find_natural · 内部跑 mdfind 'kMDItemContentModificationDate >= $time.this_week && kMDItemFSName == "*.md"'
  2. 0.1-0.5s 返回 N 个路径
  3. 按修改时间倒序展示前 10 个
  4. (可选)agent 主动问 "要打开哪个吗?"

3️⃣ 验证关键

  • 性能: 整个过程 ≤ 1 秒(mdfind 实时索引,不需要我们建)
  • 准确: 跟你跑 mdfind -onlyin ~ ... 结果对得上
  • 降级: 在 Linux 上应该走 find ~ -name "*.md" -mtime -7 不报错

4️⃣ 前置(可选)

echo "test" > ~/allhands-test/recent-doc.md && touch -d "3 days ago" ~/allhands-test/recent-doc.md

S7

一句话改权限(meta tool 核心验证)

meta.permission_grant · 三入口 SoT 验证
核心 Phase 1 3 分钟

1️⃣ 用户对话(在跑过 git push 弹了一次 confirmation 之后)

以后让你跑 git 系列命令不要再问我了

2️⃣ Agent 应该这么做

  1. 解析意图 = 给 local.bash(git:*) 加 allow
  2. 调 meta.permission_grant(pattern="local.bash(git:*)", mode="allow", scope="global")
  3. 这是 WRITE · 弹 confirmation:"我要给 local.bash(git:*) 加全局 allow · 执行?"
  4. 用户允许 → 改 ~/.allhands/permissions.json
  5. agent 回 "✓ 已添加 · 后续 git 直接放行"
  6. 用户再说 "git status 跑一下" → 不再弹

3️⃣ 三入口 SoT 验证

改完后,Settings → Permissions 页应该看到:Allow 列多了一条 local.bash(git:*)。

反过来在 UI 把它拖到 Deny 列,agent 下次 git status 就该 deny。

再 vim 编 ~/.allhands/permissions.json 把它删掉,UI 应自动 hot-reload(mtime 监听)。

S9

Proactive 触发器

独立 daemon · 文件 watcher 自动起对话
Phase 8 5 分钟

1️⃣ 配置触发器

在 Settings → Triggers 页加一条 · 或直接编辑文件:

# ~/.allhands/triggers/dl-xlsx.yaml
name: "downloads-xlsx-archive"
trigger:
  type: fs
  path: ~/Downloads
  pattern: "*.xlsx"
  event: created
employee: archiver
prompt: |
  新文件 {{event.path}} 出现在 Downloads ·
  按文件名规则归档到 ~/allhands-test/archive/
permission_profile: paranoid
notify_on_completion: true

2️⃣ 触发它

touch ~/Downloads/test-trigger.xlsx

3️⃣ 期待发生

  1. 200ms-1s 内系统通知弹出 "archiver 员工开始处理 test-trigger.xlsx"
  2. web UI 自动新建一个会话 · 你能点开看 trace
  3. 触发会话强制 paranoid profile · 关键操作仍弹通知
