🔬 10 轮迭代详解

迭代 1 · 横扫业界本地 AI agent 方案

行业对标 · 找抄什么 · 找避什么

调研发现

  • OpenHands(同名提醒!)68k star · Docker runtime + Plugin 系统(Jupyter/VSCode/AgentSkills)· 有 Local + Docker workspace 抽象
  • Cline 58k · VS Code 插件 · per-category auto-approve · YOLO mode · .clinerules
  • Aider 41k · CLI · 每改自动 git commit
  • Goose 32k · Rust + Electron 桌面 app · 3000+ MCP server 接入 · 70+ 内置 ext · 已捐 Linux Foundation
  • Continue 31k · BYOM · 配置 YAML · 偏代码补全
  • Claude Code · settings.json 三态 · /loop 自调度 · skill 体系 · Computer Use(2026.3 GA)

v1 缺位

  • v1 8 个 tool 跟 OpenHands 30+/Cline 15+/Goose 70+ 差距太大
  • 没有 Plugin/Extension 系统 · 不能让用户 "装个能力包"
  • Cline 的 per-category auto-approve 比我们"每条规则一弹"人性多了
  • Goose 的"对话里渲染交互 UI"(buttons/forms)体验比纯文本好
  • Codex 是行业唯一"sandbox 默认开"的

V2 调整

  • 新增 §1 整章"行业对标"作脚手架
  • tool 数从 8 → 40+ · 分 6 tier(借鉴 Computer Use 的 connector→GUI 分层)
  • permission 加第 4 级 "Tool Category Auto-Approve"(抄 Cline)
  • Sandbox 默认从 off → light(抄 Codex workspace-write)
  • Skill 体系扩展为"知识包 · 能力包(capability pack)"两层
Sources: OpenHands Runtime Cline auto-approve Goose docs awesome-cli-coding-agents

迭代 2 · 定义"全知全能"能力矩阵

把"无所不能"拆成 12 个可枚举能力域

调研发现

  • v1 一直没明确"全知全能"具体是哪些能力 · 只列了 5 场景
  • 对比同行 · 通用能力域应该是 12 类:文件 / 进程 / 代码语义 / 包管理 / 网络 / OS 自动化 / GUI / 浏览器 / 数据源 / 通信 / AI 服务 / 自我扩展
  • Cline 不做 OS 自动化 · Goose 靠 ext · 这是差异化空间

v1 缺位

  • 无 OS 自动化(macOS Shortcuts / Win COM / Linux D-Bus)
  • 无代码语义层(LSP/AST · 只能 grep)
  • 无统一包管理(brew/apt/pip/npm 都靠 bash)
  • 无 AI 服务路由(给 agent 调本地 whisper/embedding)

V2 调整

  • 新增 §2 能力矩阵表 · 12 域 × v1 / V2 / 落点 · 作设计脚手架
  • 每个 tool 标 tier(0-5) · 落进 §3 6 tier 架构
  • 跟 5 同行做"全行业对比矩阵"(§17)· 标差异化高地

迭代 3 · OS 原生自动化

macOS / Win / Linux · 怎么"指挥"系统

调研发现

  • macOS:AppleScript via osascript · Shortcuts CLI(shortcuts run "Name") · 受 TCC(Automation/Accessibility)权限制约
  • Win:PowerShell · COM 自动化 · UI Automation API
  • Linux:D-Bus(KDE/GNOME 通用) · xdotool
  • 关键:Shortcuts 不需要 Accessibility 权限(用户自建的会用 osascript 才会触发)

v1 缺位

  • v1 完全没考虑发 iMessage / 加日历 / 发 Mail / 弹通知这类系统级操作
  • 用户问"提醒我 3 点开会"会失败 · 这是基本盘

V2 调整

  • 新增 Tier 2 · 8 个 os.* tool
  • macOS 优先(覆盖 95% 用户):os.osascript · os.shortcut.run · os.notification · os.imsg.send · os.mail.send · os.calendar.add
  • Win/Linux 给 stub · 友好 NotImplementedError
  • R1 风险:首次需要 Accessibility 时一次性引导(参考 Anthropic Computer Use 引导)
Sources: macOS Accessibility & Automation macOS TCC deep-dive

迭代 4 · Computer Use / 浏览器 / GUI 控制

视觉操作任意 app · 最后兜底

调研发现

  • Anthropic Computer Use 2026.3 GA · 截屏 + mouse/keyboard · 有 connector 优先,GUI 兜底
  • 每次截屏 ~3K tokens · 成本是 Tier 0-2 的 10-100 倍
  • 需要 macOS Accessibility 权限 · 用户首次启用要引导
  • Playwright 是浏览器场景下更合适的工具(headless · 快 · 准)

v1 缺位

  • v1 完全没有 GUI 操作能力 · 用户问"在 Notion 里建个页面"无法做
  • 没有 token 成本意识 · 即使加了 Computer Use,也容易让 agent 滥用

V2 调整

  • 新增 Tier 3 · 7 个 gui.* + browser.* tool
  • system prompt 里写死"Tier 优先级 5→4→2→1→0→3" · GUI 是最后选项
  • R2 风险:UI 上长任务实时显示 token 用量
  • D12 默认 OFF · 用户在 Settings 显式 opt-in 才启用
  • R9:首次启用要 1 步演示("我点桌面 Finder · 你看截图")建立信任
Sources: Computer Use API Anthropic Announce

迭代 5 · MCP 生态融入

作为 host · 接通 3000+ server

调研发现

  • 官方 registry 在 registry.modelcontextprotocol.io · 10k+ servers
  • Top installs:Filesystem 485k · GitHub 398k · PostgreSQL 312k · Brave Search 287k
  • Stripe / Supabase / Vercel / Red Hat 都有官方 server
  • Goose 接 70+ extensions · 是 MCP 集成度最深的客户端
  • Claude Desktop / Code / Cursor 共享同一份 mcp.json 格式

v1 缺位

  • v1 完全没有 MCP 概念 · 用户想接 GitHub/Slack/Notion 都得我们一个一个写
  • 错过整个生态红利

V2 调整

  • 新增 Tier 4 · 我们做 MCP host · 用 Python SDK
  • 配置文件 ~/.allhands/mcp.json 兼容 Claude/Cursor 格式 · 用户能直接搬
  • UI Settings → MCP tab 集成 registry 浏览 + 一键装
  • 每个 MCP tool 自动包成我们的 Tool · 共用同一 PermissionGate
  • D13:预装 github / postgres / brave-search 三个 · 都需要 token 才激活
Sources: MCP Registry Python SDK

迭代 6 · 长期记忆与上下文

agent 别再健忘 · Mem0 / Spotlight 双管齐下

调研发现

  • Mem0 是 2026 标准:Postgres 长期事实 + episodic 总结 · 26% 准确率提升 · 91% 延迟降 · 90% token 减
  • Letta 三层模型:core(RAM-like 永远在 context)· archival(向量库 · 显式 search)· recall(对话历史 · 按需搜)
  • sqlite-vec 2025.1 GA · 50KB extension · 单文件 · 不引入外部 DB
  • macOS mdfind 已经索引 125+ 元数据 · 文档全文 · 邮件 · 联系人 · -interpret 支持自然语言

v1 缺位

  • v1 agent 完全无记忆 · 跨会话 0 复用 · 用户"上次怎么处理 sales.xlsx"问到只能再说一遍
  • 没有本地知识库 RAG · 用户全机的文档对 agent 不可见

V2 调整

  • 新增 §6 三层记忆 · core(注 system prompt)+ recall(SQLite FTS5)+ archival(sqlite-vec)
  • 5 个 memory.* tool · 含 GDPR-style memory.forget
  • 新增 local.find_natural 包装 mdfind · macOS 上零成本 RAG
  • 降级:Linux locate + recoll · Windows Search COM
  • R6 风险:UI 显示存储占用 · 能"清理 30 天前的 archival"
Sources: Mem0 长期记忆 2026 记忆框架对比 mdfind for AI

迭代 7 · Proactive / Always-on

让 agent 主动起来 · 不只是 user 开窗才动

调研发现

  • OpenClaw / Codex 都有 Heartbeat 模式让 agent 持续 alive
  • Task Trigger 模式:cron + file event 触发 · 200ms 响应
  • "Adaptive scheduling" - agent 自己设 next wakeup time(很像我们的 ScheduleWakeup)
  • Python 生态:watchdog(fs)+ apscheduler(cron)+ aiohttp(webhook)三件套

v1 缺位

  • v1 只会被动响应 · 用户"晨报推到 Slack"做不了
  • "~/Downloads 来新文件就归档"这种自动化无法实现

V2 调整

  • 新增 §8 Proactive 层 · 独立 daemon process
  • 触发器配置 ~/.allhands/triggers/*.yaml · fs/cron/webhook 三类
  • 每个 trigger 指定员工 + prompt + permission_profile
  • Settings → Triggers tab CRUD + 历史
  • R4:trigger 默认 max_runs_per_hour: 5 · 强制 paranoid profile · 防跑飞
  • D15:daemon 跟 backend 不同生命周期(backend 关 · daemon 仍跑)

迭代 8 · 安全模型重构

5 级护栏 · 默认更安全 · 老用户体验不退化

调研发现

  • OpenAI Codex 是行业唯一"sandbox 默认开" · workspace-write 模式 · 很合理
  • Cline 的 per-category auto-approve(read/edit/exec/browser/mcp 各自独立 toggle)极人性
  • Codex 加"trusted/untrusted workspace" · untrusted 时跳过项目级 .codex/ 配置(防恶意目录注入)
  • Cline YOLO mode 给 advanced 用户"我懂风险一键全开"

v1 缺位

  • v1 只有"3 态 pattern" · 不够灵活 · 也不够安全
  • 没有 workspace 信任概念 · 把项目目录当成 ~/ 同等危险
  • 没有 YOLO 出口 · 高级用户只能手 allow 每条

V2 调整

  • §5 重写权限模型 · 5 级:
  • ① YOLO mode(全开 · 红条提醒)
  • ② Sandbox boundary(off / light(默认) / strict)
  • ③ Workspace trust(trusted / untrusted · untrusted 跳过项目级配置)
  • ④ Pattern 规则(v1 三态 · 保留)
  • ⑤ Auto-approve category(5 个 toggle · 抄 Cline)
  • D11 sandbox 默认 light · D14 untrusted 处理
Sources: Codex sandbox docs Cline auto-approve Sandbox deep dive

迭代 9 · Tool / Skill 自合成

让 agent 自我扩展 · 把成功流程变 skill

调研发现

  • anthropics/skill-creator 是官方 pattern · "完成长任务后,问 agent 要不要把刚学的封成 skill"
  • SKILL.md universal 标准:Claude Code · Cursor · Codex · Gemini CLI · Antigravity 全兼容
  • VoltAgent/awesome-agent-skills 已经 1000+ 社区 skill
  • Goose 的 "recipes" 是另一种实现 · 但偏静态模板,不如 skill 灵活

v1 缺位

  • v1 skill 全是我们事先写好的 5 个 · 用户场景一变就要等我们出新版
  • 没有"沉淀经验"机制 · 同一个流程跑 N 次每次都重新教 agent

V2 调整

  • §7 全章新增 Skill Self-Synthesis
  • meta.create_skill(scope, name) tool · 套 skill-creator 模板生成
  • 3 种触发:会话末 ghost button / agent 主动建议(连续 3 次相似任务) / 用户明令
  • 沉淀位置 ~/.allhands/skills/learned/ · D14 自动 git init · 用户能 push GitHub 跟同事分享
  • 实验级 Phase 10:meta.create_tool 让 agent 直接造 backend tool(高风险 · 默认 disable)
Sources: anthropics/skills agent-skill-creator awesome-agent-skills(1000+)
