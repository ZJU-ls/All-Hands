2核心三件套

T

Local Tool

8 个 backend tool · 直接在宿主进程跑 · 流式 stdout · cwd 自动校验。完全对齐 Claude Code 的 Bash/Read/Write/Edit/Glob/Grep。

P

JSON 文件 + 三态(allow/ask/deny)+ deny>allow>ask 优先级。三层覆盖(全局 / 项目 / 会话)。兼容 Claude Code settings.json 格式 · 一键导入。

S

Skill Pack

5 个内置 Skill 教 agent 干活的方法。复用现有 ADR 0015 三段 lazy-load。新场景 = 加一个目录,零代码改动。

3权限系统(全套示范)

配置文件结构 · ~/.allhands/permissions.json

{
  "permissions": {
    "default_mode": "ask",
    "allow":  ["local.read", "local.list_dir", "local.glob", "local.grep",
               "local.bash(git status)", "local.bash(ls:*)"],
    "ask":    ["local.write", "local.edit", "local.bash(npm install:*)"],
    "deny":   ["local.bash(rm -rf /*)", "local.bash(sudo:*)"]
  },
  "workspace": {
    "default_cwd":   "~/allhands-workspace",
    "allowed_roots": ["~/allhands-workspace", "~/Documents/work"]
  },
  "sandbox": { "mode": "off", "deny_network": false }
}

规则匹配可视化

Tool 调用 → Permission Resolver 匹配规则 → 按 deny > allow > ask 优先级落到一条 lane:

✅ ALLOW · 直接执行

local.read local.bash(git status) local.glob

⚠️ ASK · 弹 Confirmation

local.write local.edit local.bash(npm install:*)

⛔ DENY · 立即拒绝

local.bash(rm -rf /*) local.bash(sudo:*)

三层覆盖

层级路径用途
全局~/.allhands/permissions.json所有 workspace 默认规则
项目级<workspace>/.allhands/permissions.json覆盖全局 · 仅本 workspace
会话内内存(可选落盘)用户在 UI 临时改 · 关页面就没
运行时: 复用现有 ConfirmationGate(L4 护栏)+ DeferredSignal 链路(ADR 0018)· PermissionResolver.decide(tool_id, args) → Allow / Deferred(ask) / Deny。

4无感引导(零配置启动)

设计目标: 用户首次启动不需要懂"权限规则",直接对话就能干活。所有引导自然嵌在交互里。

首次启动 3 步向导(全部可跳过)

选个工作目录(可拖拽 / 输入路径)· 默认 ~/allhands-workspace
选个权限策略 profile(三选一 · 默认 balanced)
跑 setup-local-env.sh(按钮一键 · 装 Python 标准库)

3 个内置 Profile

Profile适合default_mode关键差异
trusted 自己机器 · 高度信任 agent allow 只 deny 高危(rm -rf · sudo · curl|sh)· 其余全过
balanced 默认 大部分用户 · 平衡 ask READ 默认 allow · WRITE 默认 ask · 高危 deny
paranoid 敏感目录 · 多重保险 ask 全部 ask(连 read 都问)· 沙箱 mode=light

进入主界面后的隐式引导(不打断)

1

第一次 confirmation

弹窗多 3 个按钮 · 允许这一次 · 总是允许这种 · 总是允许这个 tool · 点"总是" = 自动写一条 allow 规则

2

同类弹问第 2 次

弹窗顶部加提示条:"这是第 2 次问 X · 要不要免问?" + 一键添加 allow

3

设置页 Pill 推荐

右上角 Pill:"本月被弹了 N 次确认 · 看建议" · 点开列出"高频弹问"建议批量加 allow

5Agent 一句话配权限(Meta Tool)

设计目标: 不用 UI 就能改权限。用户对话一句"以后 git push 直接放行" → Lead Agent 调 Meta Tool 改 permissions.json · 即时生效。

3 个 Meta Tool(scope=WRITE · 改完弹一次确认)

allhands.meta.permission_grant WRITE
入参:pattern · mode (allow/ask/deny) · scope (global/workspace/session) · remember=true
allhands.meta.permission_revoke WRITE
入参:pattern · 删除匹配规则
allhands.meta.permission_test READ
入参:tool_id · args · 给 Agent 自检"这条命令会被怎么处理"

典型对话(用户感受)

用户: 以后让你跑 git 系列命令,不用每次问我了
Lead: 好的,我准备给 local.bash(git:*) 加一条 allow 规则
(作用域:全局 · 永久生效)
要执行吗? [允许] [拒绝]
用户: 允许
Lead: ✓ 已添加。后续 git status / git diff / git push 等不再确认
统一 SoT: ConfirmationGate 上的"总是允许"按钮 · UI Settings 页 · Agent 调 Meta Tool · 三个入口走同一通道 · 永远改同一份 permissions.json。

6Local Executor Tools(8 个)

local.bash IRREVERSIBLE
跑 shell · 流式 stdout · 默认 60s timeout
local.read READ
读文件 · offset/limit · ≤10MB
local.write WRITE
创建 / 全量重写文件
local.edit WRITE
字符串替换 · 唯一性检查
local.list_dir READ
列目录(过滤 .git 等)
local.glob READ
glob 匹配 · 支持 **/*
local.grep READ
ripgrep 包装 · 大仓也快
local.run_python IRREVERSIBLE
跑 Python 段 · 用 ~/.allhands/venv
cwd 守护: 每个 tool 入参可带 cwd? · 默认 permissions.workspace.default_cwd · cwd 必须在 allowed_roots 内 → 否则 Deny(规则系统外的硬约束 · 防误配)。

8Python 执行环境

方案姿势优 / 劣
系统 Python 用户自己装 最简 · 缺点:污染系统
推荐 setup script uv venv ~/.allhands/venv 隔离 · 不动用户 Python
高级 用户自管 conda / pyenv 需配置 · permissions.workspace.python_path

setup-local-env.sh(一次性装好标准库)

#!/usr/bin/env bash
uv venv ~/.allhands/venv --python 3.12
~/.allhands/venv/bin/pip install \
  pandas openpyxl xlsxwriter \
  pypdf pdfplumber python-docx \
  numpy requests httpx beautifulsoup4 lxml pillow
echo "Done. allhands Python at ~/.allhands/venv"

11分阶段实现 · 5 个 Phase

  • 8 个 local.* tool + 3 个 meta tool(permission_grant / revoke / test)全部注册
  • 3 个 profile 模板(trusted / balanced / paranoid)+ 首次启动复制
  • 单元 + 集成测试 · 含 "以后 git push 直接放行" 一句话对话验证
DoD:
  • 对话让员工 "ls 我 home" 成功 · "rm 任意文件"会弹确认
  • 对话 "以后 git push 直接放行" → 改完 ~/.allhands/permissions.json → 下次 git push 不再问(无 UI 也能完成全流程)

Phase 3 · 无感引导 + 可视化权限页

⏱ 2 天

13落地后的用户视角

13.1 零配置上手(典型新用户)

安装 allhands · 3 步向导全部一键 Next · 默认工作目录 ~/allhands-workspace · 默认 balanced profile
后台自动跑 setup-local-env.sh · 装 Python 标准库
创建员工 · 默认带 local.* + 5 个 builtin skill(零额外配置)
对话:"把 sales.xlsx 里两个 sheet 的客户编号差异列出来"
agent 激活 excel-ops skill → local.read 探查 → local.run_python 跑 openpyxl
弹 confirmation 三按钮:[允许这次] [总是允许这种] [总是允许 local.run_python] · 用户点"总是允许这种" → 返回差异 + 后台静默写一条 allow 规则
同类对话再发一次 → 不再弹 · 直接出结果
全程没打开过 Settings 页。

13.2 用对话调权限(进阶 / 偷懒党)

用户: "刚才 git status 弹了三次了 · 以后所有 git 命令都别问"

13.3 高阶 / 直接编 JSON

~/.allhands/permissions.json 是单一 SoT · 用户可以直接 vim 编辑 · 文件 mtime 监听自动 hot-reload。
UI 改 / Agent 改 / 手编 — 三入口语义等价,改的都是同一份文件。
跟 Claude Code 的体感几乎一样,但多一层:可以让 agent 一句话改权限(Claude Code 现在也只能编 settings.json)。

14明确不做的事

不做为什么
❌ Docker / 容器 / workspace 实体对齐 Claude Code · 你的场景不要
❌ 多租户隔离单用户自部署
❌ 远程 worker / 任务队列本地同步执行 · 长任务靠 spawn subagent
❌ 自己实现 sandbox-exec 等价物用系统现成的
❌ pip 包管理 UI让用户跑 setup script · 不重复造轮子
❌ 文件版本控制用户工作目录大概率本来就是 git · 让 agent 用 local.bash git ...
