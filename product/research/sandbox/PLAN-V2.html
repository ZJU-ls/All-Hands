5权限模型 V2 · 五级层次

v1 是简单 JSON 三态。V2 加 4 个维度让"日常零打扰 + 关键场景强守门"成为可能。

决策合流图(每次 Tool Call 走一遍)

1
YOLO mode 是否开?
是 → ALLOW(全开)
2
触碰 Sandbox 边界?(写域 / 网络 / syscall)
越界 → DENY
3
当前 workspace 信任级别?
untrusted + WRITE → 强制 ASK
4
Pattern 规则命中?(deny > allow > ask)
→ 命中决策
5
Auto-approve category 开?(read/write/exec/browser/mcp)
cat=allow → ALLOW
6
都没命中 → 走 default_mode
默认 ASK

5 个维度详解

层级来源设置
① YOLO mode抄 Clineyolo: true · UI 顶部红条提醒
② Sandbox boundary抄 OpenAI Codexoff / light(默认) / strict · D11
③ Workspace trust抄 Codextrusted / untrusted · untrusted 跳过项目级 .allhands/ 配置
④ Pattern 规则v1 沿用deny > allow > ask · glob 匹配
⑤ Auto-approve category抄 Clineread/write/exec/browser_gui/mcp 5 个 toggle · max_calls_before_recheck

层容量何时用实现

🌟 Spotlight 作"零成本 RAG"(macOS 关键创新)

mdfind 已经索引用户全机 125+ 元数据 + 文档全文 + 邮件 + 联系人。我们不复制这个索引,给 agent 一个 local.find_natural 调它就行。

例:"找我上个月跟客户 X 讨论合同的所有文件"
→ local.find_natural("kMDItemContentModificationDate >= '2026-03' && '客户 X' && 合同")
→ mdfind 0.1s 返回 N 个路径 → agent.local.read 重点几个 → 答用户。

比维护自己的 RAG 索引便宜两个数量级。

降级:Linux locate + recoll · Windows Search COM。

7Skill 自合成(Self-Synthesis)

灵感:anthropics/skill-creator 工作流 —— "完成长任务后,问 agent 要不要把刚学的封成 skill"。

3 种触发场景

  1. 会话结束时: UI 在对话末尾显示 ghost button "把这次的方法封成 skill"
  2. agent 主动建议: 连续 3 个相似任务后 · Lead Agent 在 system prompt 收到 hint "你看起来重复做同一类事 · 提议 user 封 skill"
  3. 用户明令: "把刚才操作 sales.xlsx 的方法保存成 skill 叫 monthly-rec"

沉淀位置

~/.allhands/skills/
├── builtin/        # 我们随产品发的(excel-ops · file-organize · ...)
└── learned/        # 用户自己沉淀的 · git-managed
D14: learned/ 默认 git init · 用户能 push 到自己的 GitHub 跟同事分享。这就是 anthropics/skills repo 的私人版本。

实验级 · Tool 自合成(Phase 10 · 默认 disable)

更激进:让 agent 直接造 backend tool(meta.create_tool)。风险显著(自动生成代码注入)· 默认 disable · 只在用户开启 "Tool Synthesis" 实验开关后启用。

8Proactive 层 · 让 agent 主动起来

3 种触发器

触发器例实现
File watcher监听 ~/Downloads/*.xlsx · 一掉就让"对账员工"分类归档watchdog
Cron每天 8:30 让"晨报员工"发昨日总结到 Slackapscheduler
WebhookGitHub 给 PR 推 webhook · 触发"reviewer 员工"aiohttp

trigger.yaml 示例

name: "downloads-xlsx-archive"
trigger:
  type: fs
  path: ~/Downloads
  pattern: "*.xlsx"
  event: created
employee: archiver
prompt: |
  新文件 {{event.path}} 出现在 Downloads · 按文件名规则归档到合适目录。
permission_profile: trusted
notify_on_completion: true
R4 风险防跑飞: trigger 默认带 max_runs_per_hour: 5 · 触发跑的会话强制 paranoid profile · 关键操作仍弹通知确认。

独立 daemon process(D15)

allhands-daemon 是单独的 Python process · supervisor 起 · 跟 backend 不同生命周期(backend 关 · daemon 仍跑)· 这样"晨报"才真能晨报。

9MCP Host 实现

走 MCP Python SDK · 我们是 host(client 角色)。每个用户配置的 MCP server 在独立子进程跑 · stdio / HTTP+SSE 都支持 · 进程崩溃自动重启。

配置文件(兼容 Claude Desktop / Code / Cursor)

// ~/.allhands/mcp.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "..." }
    },
    "postgres": {
      "command": "uvx",
      "args": ["mcp-server-postgres", "postgresql://..."]
    }
  }
}

UI 一键装

Settings → MCP tab → "Browse Registry" 拉 registry.modelcontextprotocol.io 列表 → 卡片显示 description / install count → "Install" 写 mcp.json · 重启子进程。
