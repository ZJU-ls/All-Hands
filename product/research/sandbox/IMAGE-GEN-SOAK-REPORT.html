② 真实问题:wanx 单边 ≤ 1440(已修)

首轮 soak 发现两个失败: 1024x1536 和 1536x1024 在 4-5 s 后被 DashScope 拒, 错误 Either width or height should be between 512 and 1440。 我们 core/image.ALLOWED_SIZES 通用 whitelist 包含 1536/2048, 但 DashScope wanx 实际限 每边 [512, 1440](OpenAI/Imagen 没这限制)。

修复:adapter 内 pre-flight 校验 + 结构化错误(ADR 0021)

if _validate_wanx_size(request.size) is None:
    raise ImageProviderError(
        f"size {request.size!r} unsupported by wanx (each side must be 512-1440)",
        field="size",
        expected="each dimension in [512, 1440] · try 1024x1024 / 1024x1440 / 1440x1024",
        received=request.size,
        hint="DashScope wanx caps each dimension at 1440. The ALLOWED_SIZES "
             "values 1536x1024 / 1024x1536 / 2048x2048 are valid for "
             "OpenAI/Imagen but not DashScope · pick a smaller size.",
    )

位置:backend/src/allhands/execution/model_gateway/adapters/dashscope_image.py
回归:tests/unit/test_dashscope_image_adapter.py::test_size_above_wanx_cap_rejected_with_hint(httpx mock 拦截 · 验证不会再发出请求)
省钱:之前每次错配 size 浪费 4-5 s 的 polling + 一次 task 创建配额;现在 0 网络往返。
