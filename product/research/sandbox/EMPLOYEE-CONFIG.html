员工配置指南 Claude 同款

2026-04-28 · 怎么配员工才能像 Claude Code 那样整理桌面 / 建代码库 / 处理 PDF
核心机制: 员工的"能力"= 工作区(Workspace)(去哪里干活)+ Skill(怎么干 · 知识 + 工具集)。
你不需要直接挑一个个 tool —— 选对 skill · 工具就自动带上了。整个体验跟 Claude Code 完全一致(因为我们就是抄它的)。

13 步通用配置(适用所有场景)

1

配 1 个工作区(只做一次)

📍 在 web UI:Settings → Workspaces · /settings/workspaces

员工的所有文件操作都被锁在工作区根目录内。不配工作区 · local-files skill 的 7 个工具会全部拒绝执行(平台层硬约束 · 安全)。

例: 整理桌面 → 工作区 root_path = ~/Desktop 或 ~/Documents;建代码库 → root_path = ~/code · ~/projects 这种你专门留给 agent 干活的目录。

每个工作区还能勾 read_only(只读模式)+ 设 denied_globs(禁这些 pattern · 比如 **/.env)。

2

创建员工 · 选 skill(自动带工具)

📍 在 web UI:Employees → 新建员工 · /employees/new

表单只 4 个核心字段:

字段填啥
Name / Description"文件管家" / "代码工程师"等 · 给员工起个角色名
Skills(关键)多选 · 见 § 3 / § 4 推荐组合
Model推荐 Claude Sonnet 4.6 / Haiku 4.5(对长 tool-call 链路最稳定)
System Prompt角色定位 · 1-2 段 · 不要复述工具用法(skill 已经讲了)

保存 → 自动跳到员工详情页 Config tab。

3

开聊

📍 在 web UI:进员工详情 → 点 "Chat" 或主导航 → 选这个员工

说人话 · 像跟 Claude Code 对话。Agent 会:

  1. 看到 skill descriptor("本地文件 · Claude-Code 同款工具")
  2. 判断要做啥 → 调 resolve_skill 激活(注入 skill body)
  3. 按 skill 决策树调具体 tool(list_directory · bash · read_file · ...)
  4. 每个 WRITE / IRREVERSIBLE 操作前弹 Confirmation · 点"允许这一次"或"总是允许这种"

2当前可用的所有 Skill(选什么)

Skill ID能干啥带哪些 tool用于场景
allhands.local-files 主力 读写代码 · 跑 shell · 找符号 · 改 bug · 建项目 · 整理目录 · 执行 skill 脚本 list_directory · glob · grep · read_file · write_file · edit_file · bash · run_skill_script 整理桌面 · 建代码库 · 改 bug · 跑测试

核心 = allhands.local-files。这一个就给员工 8 个 Claude-Code 同款工具 +跑 anthropic 脚本的能力。其余按需求叠加。

3场景 A · 文件管家(整理桌面)

"桌面管家" 员工配置

让 Agent 帮你整理 ~/Desktop 文件 · 按类型 / 时间 / 主题归档
Workspace
先去 Settings → Workspaces 加一个:
name: 桌面 · root_path: /Users/<you>/Desktop · read_only: false
注意 桌面有些隐藏文件(.DS_Store 等)· 可加 denied_globs: ["**/.DS_Store"]
Name
桌面管家
Description
"按文件类型 / 修改时间整理桌面文件 · 定期清理临时文件"
Skills
allhands.local-files anthropic.pdf anthropic.xlsx anthropic.docx
local-files 是核心(读 / 写 / mv / mkdir / glob)· anthropic.* 让你需要时(发现 .pdf / .xlsx / .docx)能直接抽内容判断怎么归档。
Model
claude-haiku-4-5(便宜 + 工具调用稳)
System Prompt
你是桌面文件管家。

核心原则:
1. 先 list_directory 看现状 · 再列计划 · 用户同意后再动手
2. 移动 / 删除前永远先告诉用户"我准备做 X"
3. 用 mv 而不是 cp+rm(可逆)
4. 删除走系统垃圾桶 · 不要 rm -rf
5. 完成后给一句话总结(移了 N 个 · 删了 M 个 · 释放多少空间)

4场景 B · 代码工程师(建代码库)

"全栈工程师" 员工配置

让 Agent 像 Claude Code 一样建项目 · 改代码 · 跑测试 · 提交 git
Workspace
name: 代码 · root_path: /Users/<you>/code · read_only: false
或者每个项目一个 workspace:~/code/myproj
Name
全栈工程师
Description
"按需求建项目 · 写代码 · 跑测试 · 维护 git history"
Skills
allhands.local-files allhands.script_demo
local-files 提供完整 read/write/edit/bash/grep/glob 工具集。
script_demo 演示如何把"对账"等工作流封成可重复跑的脚本(后续可换成你自己的)。
Model
claude-sonnet-4-6(代码任务推荐 · 上下文长 · 推理强)
System Prompt
你是工程师 · 跟 Claude Code 风格一致工作。

工作循环:
1. 探(grep / glob / list_directory)
2. 看(read_file 读上下文 · 改之前先读)
3. 改(edit_file 精确替换 · write_file 整文件覆盖)
4. 验(bash 跑测试 / 跑构建)
5. 收尾(git status · 必要时提交)

约束:
- 每步小 · 改完立刻 bash 验
- TDD · 先写失败测试 · 再写实现
- commit message 写"为什么" · 不只是"什么"
- 不擅自 git push · 用户明令才 push

7立刻验收 · 5 分钟跑通流程

  1. 启动 backend + frontend(uv run uvicorn ... + pnpm dev)
  2. 浏览器开 web UI · 点 Settings → Workspaces · 加一个 ~/allhands-test(先 mkdir ~/allhands-test · 安全)
  3. 点 Employees → 新建 · 起名"测试管家"· 选 skill allhands.local-files + allhands.script_demo · 模型 claude-haiku · 保存
  4. 跳到员工详情 → Chat
  5. 说:"跑一下 echo.py · args 是 ['你好', '早安']"
  6. 看到:Agent 调 resolve_skill(allhands.script_demo) → 弹 Confirmation 跑 run_skill_script → 你点允许 → 输出 "你好 早安"
跑通 = 全链路打通(workspace + skill + tool + run_skill_script + Confirmation Gate)= 你可以信心满满推 main 给同事用了。
