图片生成能力 · 调研 / 设计 / 实现 2026-04-28

让员工能在生成 PPT/Markdown 时异步插入 AI 生成的图片 · 而不是只能写文字
核心结论: 不能。当前平台:
  • ✅ artifact 系统已支持 kind: image + pptx 已支持 slide 内嵌图(image_url data URL)· markdown / html 也能内联
  • ❌ 没有图片生成 tool(grep 全代码 0 个 image generation 调用)
  • ❌ Provider 模型只识别 chat(ProviderKind = "openai" | "anthropic" | "aliyun"· 都按 chat 算)· LLMModel 没有 capability 字段区分图/语音
  • ❌ 没有异步任务机制:image gen 单张 5-30 秒 · 阻塞 agent loop 体验崩
差 4 个能力 · 但都是渐进可加 · 不破坏现有架构。 见 § 3。

1现状深扫(基于代码 audit)

🖼️

图片载体系统

已就位

artifact_create(kind=image, content_base64, mime_type) 完整 · 上限 20MB · 支持 png/jpg/webp/gif/svg。pptx 的 image-right layout 接 image_url: "data:..."。markdown 用 ![alt](data:...)。

🌐

artifact URL 接入

已就位

GET /api/artifacts/<id>/content 端点存在 · 这意味着图片生成后存 artifact · markdown 用 ![alt](/api/artifacts/<id>/content) 而不是塞 base64 进文档(节省 token + 文档体积)。

🎨

图片生成能力

完全缺

全代码 grep image_url · generate_image · dall-e · gpt-image · imagen · midjourney · 0 个引用(只有 anthropic 多模态 chat 的 image_url 是读图不是生图)。

⚡

异步任务机制

缺

所有 tool 调用同步阻塞 agent loop · 30 秒图生 = LLM context 等 30s 才能继续。没有 "fire-and-forget + 后续 await / poll" 模式。

🔌

Provider 模型类型

缺

LLMProvider.kind 只 3 种:openai/anthropic/aliyun · 都假定是 chat。LLMModel 没 capability 字段区分 chat/image/speech/embedding。

💰

计费 / 配额

部分

Pricing service 已有 chat 的按 token 计 · 但图片是按张计费($0.04-$0.17 per image · GPT Image 1.5 / Imagen / Flux 各异)· pricing model 未覆盖。

2用户场景拆解 · 你想做什么

诉求原文: "画 PPT 或者生成 markdown 的时候 · 可以插入图片生成模型的图片【异步】并组织好一个带图片的 markdown 或者 ppt"

2.1 拆成 3 类操作

操作同步 / 异步预期延迟优化点
① 单张图生(用户:"给我画个 X")可同步5-15s同步等可接受 · 配 thinking 提示
② N 张图批量(PPT 8 张配图)必须并发5-15s × 8 = 串行 2 分钟 · 并发 15sfan-out · 等所有完成
③ 流式生成 markdown 边写边插图必须异步文字立即可见 · 图后填充占位符 + 后台填充

2.2 关键问题:agent loop 阻塞 vs 占位符替换

权衡:
  • 同步等(简单):tool 调用阻塞到图生成完 · agent 接到 base64 后塞 markdown · 优:实现简单 · 劣:1 个 PPT 8 张图 = 8 × 15s = 2 分钟 LLM 干等
  • 并发 fan-out + barrier(中):单 tool call 接 N 个 prompt · 内部并发起 N 个任务 · barrier wait · 一次性返回 N 个 artifact_id · agent 拿到全部后写 markdown · 推荐 v1
  • 占位符 + 后台填充(终极):agent 立即返回带 ![](pending:img-001) 占位的 markdown · 后台 task 跑完 patch · 前端 SSE 推 patch event 局部刷新 · 体验最佳但工程量最大 · 留 v2

3缺哪些基础能力(Gap 分析)

G1

图片生成 Tool · allhands.image.generate

核心 unblocker。BACKEND tool · scope=WRITE + requires_confirmation(可配 · 默认 ask · 因为图生收费)。返回 artifact_id · 自动调 artifact_create 落库。支持单 prompt 同步 + 多 prompt 并发 batch。

⏱ 1-2 天
G2

Image Provider 抽象 · ImageProvider + capability 字段

核心解耦。LLMModel 加 capabilities: list[Capability](chat / image / speech / embedding)· UI Provider 配置时按 capability 过滤可选模型。新加 OpenAI Images / 通义万相 / Stable Diffusion 直接走 ImageProvider 子类 · 不污染 chat model_gateway。

⏱ 1.5 天
G3

异步 Tool 模式 · spawn_long_task + wait_task

通用机制 · 不只为图片。Tool 启动一个后台 asyncio.Task · 立即返回 task_id · agent 后续调 wait_task 等(类似 Claude Code 的 BashOutput 模式)。给图生 batch / 长 bash / web scraping 等通用复用。

⏱ 2 天
G4

图床 / 图片 URL 解析 · artifact://<id> URL scheme

现有 /api/artifacts/<id>/content 已能取图。但 markdown 里写绝对 URL http://localhost:8000/... 不可移植 · 离线打开就坏。引入 artifact://<id> 协议:渲染时按当前 host 替换 · 导出 PPT/PDF 时把 artifact 内容 inline 化 base64 内嵌。

⏱ 0.5 天
G5

Pricing extension · 按张计费

现 pricing 按 token · image 按张(+ size + quality 维度)。Model.cost_per_image 字段 + ledger 记录。配额超时 deny · 不让 agent 跑飞烧钱。

⏱ 0.5 天

总工作量

v1 必须 4 个(G1+G2+G4+G5)= ~ 3.5 天(同步 + batch · 实用)
v1.5 加 G3(异步 task)= + 2 天 = 5.5 天(并发更优)
v2 加占位符 + SSE patch = + 3 天(终极体验)

4.1 LLMModel 加 capability 字段(L4 core)

# core/model.py
from enum import StrEnum

class Capability(StrEnum):
    CHAT = "chat"
    IMAGE_GEN = "image_gen"
    SPEECH = "speech"
    EMBEDDING = "embedding"

class LLMModel(BaseModel):
    # ... 现有字段
    capabilities: list[Capability] = Field(default=[Capability.CHAT])
    cost_per_image: float | None = None  # USD per image (image_gen only)
    cost_per_image_size: dict[str, float] | None = None  # {"1024x1024": 0.04, ...}

4.3 Tool 设计(execution/tools/builtin/image_generate.py)

IMAGE_GENERATE_TOOL = Tool(
    id="allhands.image.generate",
    kind=ToolKind.BACKEND,
    name="generate_image",
    description=(
        "Generate one or more images from text prompts via the configured "
        "image model. Returns a list of {artifact_id, mime_type, prompt} — "
        "each image is automatically saved as an artifact you can reference "
        "in markdown via /api/artifacts//content or in pptx via "
        "image_url. For batch (e.g. 8 PPT slides), pass `prompts: [...]` "
        "all at once — they run concurrently."
    ),
    input_schema={
        "type": "object",
        "properties": {
            "prompts": {
                "type": "array",
                "items": {"type": "string", "minLength": 3, "maxLength": 4000},
                "minItems": 1, "maxItems": 10,
                "description": "1-10 prompts · run concurrently",
            },
            "size": {
                "type": "string",
                "enum": ["1024x1024", "1024x1536", "1536x1024", "auto"],
                "default": "1024x1024",
            },
            "quality": {"type": "string", "enum": ["low","medium","high","auto"], "default": "auto"},
            "model_ref": {
                "type": "string",
                "description": "Override: provider/model · e.g. 'openai/gpt-image-1.5'. Default uses workspace's configured image model.",
            },
        },
        "required": ["prompts"],
    },
    output_schema={
        "type": "object",
        "properties": {
            "images": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "artifact_id": {"type": "string"},
                        "mime_type": {"type": "string"},
                        "prompt": {"type": "string"},
                        "size": {"type": "string"},
                        "url": {"type": "string", "description": "/api/artifacts/<id>/content"},
                    },
                },
            },
            "total_cost_usd": {"type": "number"},
            "duration_ms": {"type": "integer"},
            "error": {"type": "string"},
        },
    },
    scope=ToolScope.WRITE,
    requires_confirmation=True,  # 收费 · 默认 ask · 用户可改 allow
    cost_hint=CostHint(relative="medium", absolute_per_call_usd=0.05),
)

9实施完后的对话脚本(目标体验)

用户:"做一个 6 页关于'量子计算未来 5 年趋势'的 PPT · 每页配张图"
Lead Agent: 
1. resolve_skill(artifacts) + resolve_skill(image-creator)
2. 拟大纲:
   - p1: 标题页(配:量子计算抽象艺术图)
   - p2: 现状(配:IBM/Google 量子芯片实拍风)
   - p3: 突破方向(配:超导 vs 离子阱 对比示意图)
   - p4: 商用进展(配:商业服务图标拼接)
   - p5: 5 年预测(配:时间线带科技元素)
   - p6: 总结(配:简洁科技封面图)
3. 调 generate_image({prompts: [上面 6 个 prompt], size: "1024x1536"})
   → 弹 confirmation:"将调 OpenAI gpt-image-1.5 生 6 张图 · 估 $0.24 · 允许?"
   → [允许] · 后台 asyncio.gather 6 个并发 · ~ 18s
   → 返回 6 个 artifact_id
4. 调 artifact_create_pptx({slides: [
       {layout: title, title: "量子计算未来 5 年", subtitle: "...", image_url: "artifact://img-001"},
       ...
   ]})
5. 返回 PPT 给用户 · 弹 preview · 带 6 张配图

全程1 次 generate_image batch + 1 次 pptx 创建 = 2 次 confirmation 弹窗 · 整个对话 30-40s 完成 · 拿到完整带图 PPT。
