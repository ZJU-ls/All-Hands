# Cron 表达式速查 · 常用 schedule

> 用 `read_skill_file('allhands.triggers_management', 'references/cron-cookbook.md')` 拉这份 · 直接抄。

## 5-field 格式

```
* * * * *
| | | | |
| | | | └── 星期 (0-6, 0=Sun · 也支持 MON-FRI / SAT,SUN)
| | | └──── 月 (1-12 · 也支持 JAN-DEC)
| | └────── 日 (1-31)
| └──────── 小时 (0-23)
└────────── 分钟 (0-59)
```

## 高频模式(用户问「每天 9 点」就抄这个)

| 用户语言 | cron | 说明 |
|---|---|---|
| 每分钟 | `* * * * *` | 适合 dev 调试 · 生产慎用 |
| 每 5 分钟 | `*/5 * * * *` | 监控类常用 |
| 每 15 分钟 | `*/15 * * * *` | 行情拉取常用 |
| 每小时整点 | `0 * * * *` | |
| 每天早上 9 点 | `0 9 * * *` | UTC · 北京时间需 +8 |
| 每天早上 9 点(北京) | `0 1 * * *` | 北京 9 = UTC 1 |
| 每周一 9 点 | `0 9 * * 1` 或 `0 9 * * MON` | |
| 工作日 9 点 | `0 9 * * 1-5` 或 `0 9 * * MON-FRI` | |
| 每月 1 号 8 点 | `0 8 1 * *` | 月报常用 |
| 每月最后一天 | `0 18 28-31 * *` (近似 · cron 不直接支持「最后一天」) | 配合 if 判断 |
| 每季度第一天 | `0 9 1 1,4,7,10 *` | |
| 凌晨 3 点(运维窗) | `0 3 * * *` | 备份 / cleanup 常用 |

## 时区陷阱

服务端 cron 走 **UTC** · 用户给的「9 点」要先确认时区:

| 用户时区 | 9:00 本地 | UTC cron |
|---|---|---|
| Asia/Shanghai (UTC+8) | 09:00 | `0 1 * * *` |
| America/New_York (UTC-5) | 09:00 | `0 14 * * *` |
| Europe/London (UTC+0) | 09:00 | `0 9 * * *` |

## 常见坑

- **5 vs 6 字段**:有些系统用 6 字段(加秒),我们是 **5 字段** · 别抄错
- **`* * * * 0`** 和 **`* * * * 7`** 都是周日 · 但有些库不接受 7 · 统一用 0 或 SUN
- **每月 31 号** (`0 9 31 * *`) · 在没有 31 号的月份(2/4/6/9/11)直接跳过 · 不会报错也不会顺延
- **多值** 用逗号:`0 9,12,18 * * *` = 每天 9 点 / 12 点 / 18 点
- **范围** 用连字符:`0 9-17 * * 1-5` = 工作日 9-17 整点
